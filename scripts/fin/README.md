# FIN コードジェネレータ

`fin_codegen.py` は [`docs/fin/fin-domain-manifest.yaml`](../../docs/fin/fin-domain-manifest.yaml) を
単一ソースとして、FIN（経理・人事）モジュールの DB / バックエンド / フロントエンド /
メニュー / ルート / 多言語リソースを生成します。

## 依存

- Python 3.10+（`py -3` で起動可）
- PyYAML のみ（Jinja2 不要・純 Python テンプレート）

## 使い方

```bash
# 全量生成
py -3 scripts/fin/fin_codegen.py --target all

# 個別生成（カンマ区切り可）
py -3 scripts/fin/fin_codegen.py --target db
py -3 scripts/fin/fin_codegen.py --target backend,frontend
py -3 scripts/fin/fin_codegen.py --target menu,router,locales
```

| `--target` | 生成物 |
|------------|--------|
| `db`       | `backend/database/migrations/36_fin_full_schema.sql`（約 40 表 + 種子データ） |
| `backend`  | `backend/app/modules/fin/<domain>/{models,schemas,api}.py` ＋ 集約 `__init__.py` ＋ `common/crud.py` |
| `frontend` | `frontend/src/views/fin/**`（45 画面ラッパ） |
| `menu`     | `frontend/src/router/finMenuConfig.ts` |
| `router`   | `frontend/src/shared/router/finRoutes.ts`（旧 ERP 原価ページの redirect 含む） |
| `locales`  | `frontend/src/locales/fin/{ja,zh,en,vi}.ts` ＋ `index.ts` |

## 生成 / 手書きの境界

- **生成（再実行で上書き）**: 先頭に `@generated by scripts/fin/fin_codegen.py` マーカーを持つファイル。
- **手書き（再生成で上書きされない）**:
  - `backend/app/modules/fin/common/crud.py`（汎用 CRUD ビルダー。安定版）
  - `backend/app/modules/fin/services_api.py`（中核エンドポイント）
  - `backend/app/modules/fin/**/services/*.py`（posting / calc / attendance）
  - `backend/app/modules/fin/integration/erp_events.py`
  - `frontend/src/views/fin/components/*`（汎用 Vue コンポーネント）
  - `frontend/src/api/fin/index.ts`、`frontend/src/composables/useMenuTree.ts`

## 既存ファイルへの取り込み（手動・1 回のみ）

生成物は以下に **spread / merge** 済み（再生成で再編集不要）:

- `frontend/src/router/menuConfig.ts` … `...finMenuConfig`
- `frontend/src/shared/router/index.ts` … `...finRoutes`（MainLayout children）
- `frontend/src/i18n.ts` … `mergeLocale(base, finLocales.*)`
- `frontend/src/components/layout/SidebarMenu.vue` … `<MenuTreeItem :node="finMenu" />`
- `backend/app/main.py` … `fin.router` ＋ `fin_services_router`

## マニフェスト変更後の運用

1. `docs/fin/fin-domain-manifest.yaml` を編集（エンティティ列・メニュー追加など）。
2. 対応 `--target` を再実行。
3. DB はテーブル追加なら増分 `37_fin_*.sql` を手書き（`36_*` の再生成は新規構築用）。
4. 画面 / メニュー / ルート / locale は再生成で同期。
5. アプリ起動後、システム管理 →「メニュー管理 → ルート定義から取り込み」で `menus` テーブルへ反映。
