# 検査モニタ 画面 操作説明

| 項目 | 内容 |
|------|------|
| メニュー | **MES** → **モニタリング** → **検査モニタ** |
| 画面名（画面上） | **検査モニタ** |
| 操作説明画面 | 画面上部 **マニュアル** ボタン → 左メニュー **MES関連** → **検査モニタ**|
| 次製品指定 | 追加で MES **編集権限**が必要 |

本書は、検査現場の **担当者・班長・事務** が、当日の検査稼働状況を **一覧モニタリング** し、検査員へ **次製品（参考）** を指示する操作を説明します。  
検査員自身の実績入力は **[検査実績収集]**、事後登録は **[検査実績収集登録]** を使用してください。

## 1. 画面の目的とできること

検査モニタは、選択した **生産日** の検査現場を **リアルタイムに俯瞰** するダッシュボードです。

| 機能 | 説明 |
|------|------|
| 稼働監視 | 検査中・一時停止・休憩中の製品カードを一覧表示（経過時間・検査員名） |
| KPI 表示 | 稼働数、休憩、通信断、平均能率、生産数、不良数、不良率 |
| 不良モニタ | 当日発生した不良を **項目別** に一覧（検査員・製品・発生時刻） |
| 能率ランキング | 検査員ごとの平均能率・順位 |
| 確定実績 | 完了済みセッションのサマリーカード |
| **次製品指定** | 稼働中検査員の「次に検査する製品」、または未稼働検査員の「初回製品」を指定 |
| 実績登録へ | 確定実績パネルから **検査実績収集登録** 画面へ遷移 |

**表示対象データ**

- サーバー API は **MES 実績のみ** を返します（Excel/CSV 取込行はモニタに含まれません）。
- 稼働カードは **進行中**（稼働中 / 一時停止 / 休憩中）の行のみ表示します。

## 2. 画面へのアクセスと操作説明の開き方

### 2.1 検査モニタ画面を開く

1. Smart-EMAPs に **検査モニタ権限** のあるアカウントでログインします。
2. 左サイドメニュー **MES** を展開します。
3. **モニタリング** → **検査モニタ** をクリックします。

![メニューから検査モニタを開く](./images/InspectionMonitor/02_menu_navigation.png)

### 2.2 本操作説明（マニュアル）を開く

1. 画面上部ヘッダーの **マニュアル** ボタンをクリックします。
2. 新しいタブでマニュアルホームが開きます。
3. 左サイドバー **「MES関連」** → **「検査モニタ」** をクリックします。

## 4. ヘッダー（ツールバー）

![ヘッダーツールバー](./images/InspectionMonitor/03_header_toolbar.png)

| 要素 | 説明 |
|------|------|
| タイトル | **検査モニタ** |
| 時計 | 現在時刻（秒単位で更新） |
| **生産日** | 日付ピッカー。変更すると当日データを再取得 |
| **自動更新** | ON 時、約 **15 秒** ごとにサーバーから再取得 |
| **全画面** | モニタ表示を全画面化（投影・大型ディスプレイ向け） |
| **更新**（↻） | 手動で即時再取得 |
| **取得失敗** | 通信エラー時、タイトル横に赤文字表示（ツールチップに詳細） |

## 5. KPI サマリー（上部指標）

![KPI サマリー](./images/InspectionMonitor/04_kpi_row.png)

| KPI | 意味 |
|-----|------|
| **稼働中** | 生産計測中（タイマー稼働）の検査件数 |
| **一時停止** | 一時停止中の件数 |
| **休憩中** | 休憩ボタンで計測中断中の件数 |
| **通信断** | 端末からの更新が **60 秒超** 途絶えた稼働カード数（0 のとき非表示） |
| **平均能率** | **確定済み** 実績のみの加重平均（本/時）。200 未満 / 800 超は赤強調 |
| **生産数** | 当日 MES 行の生産数合計 |
| **不良数** | 不良合計（0 のときカード非表示） |
| **不良率** | 不良数 ÷ 生産数 × 100（小数第 1 位、%） |

## 6. 検査工程パネル（稼働カード）

![稼働カード](./images/InspectionMonitor/05_runtime_cards.png)

### 6.1 パネルヘッダー

| 要素 | 説明 |
|------|------|
| タイトル | **検査工程** |
| **次製品指定** ボタン（箱アイコン） | 右ドロワー「次製品指定」を開く。バッジ数字 = 指定済み件数 |
| 件数 | 本日の稼働カード数 |

### 6.2 稼働カードの内容

各カードは **1 製品 × 1 検査員** の進行中セッションを表します。

| 表示 | 説明 |
|------|------|
| カード名 | `製品CD · 製品名` |
| **経過** | 稼働経過時間（`MM:SS` または `Hh MMm`） |
| **一時停止** / **休憩** | 該当状態のとき累計時間 |
| **最終通信** | 通信断時のみ、サーバー最終更新時刻 |
| 検査員名 | 右上（稼働中・停止・休憩時） |
| ステータスタグ | 稼働中 / 一時停止 / 休憩中 |
| **通信断** タグ | 端末更新途絶時（赤） |

### 6.3 空状態

| メッセージ | 意味 |
|------------|------|
| 稼働中の検査はありません | 計画行はあるが、進行中セッションなし |
| 本日の計画はありません | 当該生産日に MES データなし |

## 7. 不良（項目別）パネル

![不良一覧](./images/InspectionMonitor/06_defect_table.png)

当日 MES 実績から、不良数量が 1 以上の **項目別** エントリを時系列（新しい順）で表示します。

| 列 | 内容 |
|----|------|
| 検査員 | 担当者名 |
| 稼働状態 | 記録時点の状態タグ |
| 製品名 | 検査製品 |
| 不良項目 | 工程別不良マスタの名称 |
| 不良数 | 数量 |
| 不良発生時刻 | 時:分 |

不良がない場合：**「本日の不良はありません」**

## 8. 検査員能率パネル

![能率ランキング](./images/InspectionMonitor/07_efficiency_table.png)

| 列 | 内容 |
|----|------|
| 順位 | 1 位〜（1〜3 位はメダル色） |
| 検査員 | 氏名 |
| 件数 | 集計対象セッション数 |
| 生産数 | 合計本数 |
| 平均能率 | 本/時（200 未満 / 800 超は赤） |

パネルヘッダー右側に **当日平均能率**（確定済みベース）を表示します。

![確定実績](./images/InspectionMonitor/08_history_cards.png)

| 要素 | 説明 |

| 経過 | 正味稼働時間 |
| 検査員 | 担当者 |

## 10. 次製品指定（担当者操作）

検査員の **[検査実績収集]** 画面に **「次の製品（参考）」** バーとして表示される指示を、本画面から行います。

![次製品指定フロー](./images/InspectionMonitor/09_next_assign_workflow.png)

### 10.1 パネルを開く

1. **検査工程** パネル右上の **次製品指定** ボタン（箱アイコン）をクリックします。
2. 右側から **次製品指定** ドロワーが開きます。

![次製品指定ドロワー](./images/InspectionMonitor/10_next_assign_drawer.png)

### 10.2 ドロワー内の区分

| 区分 | 対象 | 用途 |
|------|------|------|
| **稼働中** | 現在検査中・停止・休憩の検査員 | **次** に検査する製品を指定 |
| **初回製品（未稼働）** | まだ稼働していないが事前指定済みの検査員 | 出勤前・待機中の **最初の製品** を指定 |

**検査員を追加して指定** ボタンから、一覧にない検査員を選んで初回製品を指定できます（仕上課所属）。

### 10.3 指定ダイアログ

![次製品指定ダイアログ](./images/InspectionMonitor/11_next_assign_dialog.png)

| 項目 | 説明 |
|------|------|
| 検査員 | 対象者（追加指定時はプルダウンで選択） |
| 現在 | 稼働中の場合、いま検査中の製品 |
| 次製品 / 初回製品 | 検査対象製品（CD · 名称）を選択 |
| **保存** | サーバーへ登録（`inspection_inspector_next_assignment`） |
| **指定解除** | 既存指定を削除 |
| **キャンセル** | 変更せず閉じる |

**製品候補の条件**

- 製品 CD 末尾 `1`、名称に「加工」「アーチ」を含まない（検査実績収集と同じフィルタ）

### 10.4 検査員側での表示

指定後、該当検査員が **検査実績収集** を開くと：

- 作業エリア上部に **「次の製品（参考）」** バー
- **選ぶ** ボタンで製品プルダウンにワンタップ反映（生産中は無効）

**重要**

- 指定は **参考情報** です。自動で生産開始しません。
- 検査員は手動で製品を選んで **生産開始** する必要があります。
- 約 15 秒以内にモニタ・収集画面双方へ反映されます。

## 11. 通信断の表示

MES 端末（検査実績収集）が **約 60 秒以上** サーバーへ更新しない場合、該当稼働カードに **通信断** と表示されます。

| 条件 | 説明 |
|------|------|
| 対象 | 稼働中 / 一時停止 / 休憩中 |
| 端末ロックあり | `mes_client_instance_id` が設定された MES セッション |
| 判定 | `updated_at` から 60 秒超経過 |
| KPI | 「通信断」カードに件数表示 |

**想定原因**

- 端末のネットワーク切断（オフライン計測継続中）
- ブラウザタブのサスペンド
- 端末電源オフ

**対処**

- 検査員端末のネットワーク・画面状態を確認
- [検査実績収集]のオフライン同期・セッション復旧を参照
- 復帰後、モニタの **通信断** タグが消えることを確認

## 12. 自動更新・全画面表示

### 12.1 自動更新

| 項目 | 値 |
|------|-----|
| 間隔 | 約 **15 秒**（`INSPECTION_AUTO_REFRESH_MS`） |
| 時計 | 1 秒ごとに経過時間を再計算 |
| タブ非表示 | ブラウザタブが hidden の間は **自動取得を停止**（戻ると即再取得） |
| 手動更新 | ↻ ボタンでいつでも可能 |

### 12.2 全画面表示

1. ヘッダー右の **全画面** ボタンをクリックします。
2. ページが全画面レイアウトになり、ブラウザの全画面 API も試行します。
3. 再度クリック、または Esc で解除します。

現場の大型モニター・プロジェクターでの俯瞰表示に適しています。

## 13. 検査実績収集との連携

```text
検査モニタ（担当者）                    検査実績収集（検査員）
     │                                        │
     ├─ 次製品指定 ──────────────────────────►│「次の製品（参考）」表示
     ├─ 稼働・不良・能率を監視 ◄──────────────┤ 生産開始/停止/休憩/不良入力
     └─ 確定実績確認 ──► 実績登録画面        └─ 生産終了で確定
```

| 役割 | 使用画面 |
|------|----------|
| 班長・担当者 | **検査モニタ**（監視・次製品指示） |
| 検査員 | **検査実績収集**（計測・入力） |
| 事務・補正 | **検査実績収集登録**（事後入力） |

## 14. 権限と操作制限

| 操作 | 必要権限 |
|------|----------|
| 画面表示・データ閲覧 | メニュー `MES_MONITOR_INSPECTION` |
| **次製品指定** の保存・解除 | 上記 ＋ MES **編集権限**（`canEdit`） |
| 実績登録リンク | 登録画面のメニュー権限 `MES_ACTUAL_REG_INSPECTION` |

403 エラー時：**「検査モニタへのアクセス権がありません」**

## 16. 指標の計算方法

### 16.1 平均能率（KPI・パネル）

```
能率 = 生産数 ÷ 正味稼働時間[時間]  （本/時、整数）
```

- KPI の **平均能率** は **確定済み** 行のみで加重平均
- 能率ランキングは当日全セッション（進行中含む）を検査員別に集計

### 16.2 不良率（KPI）

```
不良率(%) = (不良数合計 ÷ 生産数合計) × 100
```

小数第 1 位表示。生産数 0 のとき `—`

### 16.3 能率異常の色

- **200 本/時未満** または **800 本/時超** → 赤色（入力ミス・異常値の目安）

## 17. よくあるメッセージと対処

| メッセージ（要約） | 対処 |
|--------------------|------|
| 検査モニタへのアクセス権がありません | メニュー権限 `MES_MONITOR_INSPECTION` を確認 |
| データの取得に失敗しました | ネットワーク・サーバー状態を確認し **更新** |
| 取得失敗（ヘッダー表示） | 同上。30 秒以内の重複 toast は抑制されます |
| 検査員を選択してください | 次製品指定で検査員未選択 |
| 次製品を選択してください | 製品プルダウン未選択 |
| 検査対象外の製品です | 検査用 CD（末尾 1）の製品を選ぶ |
| 次製品を指定しました | 正常完了 |
| 次製品指定を解除しました | 正常完了 |
| 次製品の指定に失敗しました | 権限（canEdit）・ネットワークを確認 |
| 稼働中の検査はありません | 全員完了または未開始。正常な空状態 |
| inspection_inspector_next_assignment テーブルが存在しません | DB migration 48 を実行 |

## 18. 運用上の注意

1. **監視専用画面**  
   本画面から生産開始・停止はできません。現場操作は検査員端末の **検査実績収集** で行います。

2. **次製品指定は参考**  
   自動開始しません。検査員が **選ぶ** または手動選択して開始してください。

3. **通信断の見方**  
   オフライン計測中も **通信断** と表示されることがあります。端末復帰後にタグ消失を確認してください。

4. **生産日の切替**  
   夜勤・日跨ぎでは生産日を明示的に合わせてください。

5. **投影・班前ミーティング**  
   **全画面** ＋ **自動更新 ON** で、現場全体の稼働・不良・能率を共有する用途に最適です。

6. **Excel/CSV データ**  
   モニタには **MES データのみ** 表示されます。取込実績は登録画面等で確認してください。
