| 項目 | 内容 |
|------|------|
| メニュー | **MES実績収集登録** → **検査実績登録** |
| 画面名（画面上） | **検査実績収集登録** |

## 1. 画面の目的とできること

検査実績収集登録画面では、**すでに終了した検査作業** の実績を、次の単位でサーバーに登録・更新します。

**1 行 = 生産日 × 検査員 × 製品（＋必要に応じて生産順）**

| 機能 | 説明 |
|------|------|
| 手動新規登録 | 生産日・検査員・製品・生産数・開始/終了時刻・休憩/停止・不良・備考を入力して **保存** |
| 一覧表示 | 選択した生産日の登録済み行をテーブル表示（検査員で絞り込み可） |
| 編集 | 一覧行をクリックまたは編集アイコンでフォームに読み込み、内容を更新 |
| 削除 | 権限がある場合、行を削除（MES 生産中・取込データは警告付き） |
| 能率表示 | 一覧に能率（本/時）を表示。基準外は赤色で強調 |

**対象製品の条件（システム側フィルタ）**

- 製品 CD の **末尾が `1`**（検査用製品コード）
- 製品名に **「加工」「アーチ」** を含むものは一覧から除外
- 製品名の昇順でドロップダウン表示

**検査員について（重要）**

- 検査員は **仕上課所属の有効ユーザー** からプルダウンで選択します（[検査実績収集] とは異なり、ログインユーザー固定ではありません）。
- 登録・保存操作自体は **ログイン必須** です。

## 2. 画面へのアクセスと操作説明の開き方

### 2.1 検査実績収集登録画面を開く

1. Smart-EMAPs に **MES 操作権限のあるアカウント** でログインします。
2. 左サイドメニューから **MES実績収集登録** を展開します。
3. **検査実績登録** をクリックします。

1. 画面上部ヘッダー右側の **マニュアル** ボタンをクリックします。
2. **新しいタブ** でマニュアルホームが開きます。
3. 左サイドバー **「MES関連」** → **「検査実績収集登録」** をクリックします。

|--------|------|
| **印刷**（左サイドバー下部） | ブラウザの印刷ダイアログで本書を印刷 |

## 4. 入力ステップ（①〜⑦）の概要

ヘッダー右側のカラーチップは、入力順序を示します。

| ステップ | 項目 | 新規登録時の解禁条件 |
|----------|------|----------------------|
| ① | 生産日 | 常に入力可 |
| ② | 検査員 | 常に入力可 |
| ③ | 製品名 | **② 検査員選択後** |
| ④ | 生産数（箱数・本数） | **③ 製品選択後** |
| ⑤ | 生産時間（開始・終了・休憩・停止） | 同上 |
| ⑥ | 不良（項目別） | 同上 |
| ⑦ | 備考 | 同上 |

③ 以前は、画面中央に **ロックヒント**（点線枠の注意文）が表示され、④以降のエリアは薄く表示・操作不可になります。

## 5. 実績入力フォーム

### 5.1 ① 生産日

![生産日入力](./images/InspectionActualRegistration/03_production_day.png)

| 操作 | 説明 |
|------|------|
| 日付ピッカー | `YYYY-MM-DD` 形式。変更すると **登録一覧もその日付で再読込** |
| **前日 / 翌日**（矢印ボタン） | 生産日を 1 日ずつ移動 |
| **今日** | 日本標準時（JST）の当日に設定 |

フォームの生産日と一覧ヘッダーの日付は連動します。

### 5.2 ② 検査員

- **仕上課** 所属の有効ユーザーのみ候補に表示されます。
- 検索可能なプルダウンです。
- 新規登録時、検査員を **クリア** すると、すでに選んでいた製品・数量もリセットされます。

### 5.3 ③ 製品名

- 新規：検査員選択後に製品プルダウンが有効化されます。
- 編集：製品名は **読み取り専用**（変更不可）。
- 製品変更時、④ の箱数・本数はクリアされます。

### 5.4 ④ 生産数

![生産数入力](./images/InspectionActualRegistration/04_quantity.png)

| 項目 | 説明 |
|------|------|
| **箱数** | 製品マスタの **入数（本/箱）** と連動 |
| **本数** | DB の `actual_production_quantity` に保存される正式値 |
| 入数表示 | 製品選択後、ラベル横に「入数 n 本/箱」を表示 |
| 入数未設定 | 「入数未設定（本数を入力）」と警告表示。箱数からの自動計算は不可 |

**連動ルール**

- **箱数を入力** → 本数 = 箱数 × 入数（自動計算、本数側が派生表示）
- **本数を入力** → 箱数 = 本数 ÷ 入数（四捨五入、箱数側が派生表示）
- 派生側のセルは **破線枠** で区別されます

本数が入数で割り切れない場合、入力中に警告が表示され、**保存時に確認ダイアログ** が出ます（[§10](#10-生産数箱数本数のルール)）。

### 5.5 ⑤ 生産時間

![生産時間入力](./images/InspectionActualRegistration/05_time.png)

| 項目 | 形式 | 説明 |
|------|------|------|
| **開始** | `HH:mm`（例: `08:30`） | ① 生産日と合成して開始日時を生成 |
| **終了** | `HH:mm`（例: `17:00`） | 同上 |
| **休憩** | 分（0〜999） | 計画的な休憩時間 |
| **停止** | 分（0〜999） | 段替・待ち等の停止時間 |

**入力のヒント**

- `830` → `08:30`、`1730` → `17:30` のように数字のみでも解釈されます。
- **Enter キー** で次フィールドへ移動：開始 → 終了 → 休憩 → 停止 → 不良（先頭項目）
- 開始・終了が有効なとき、下部に **シフト時間・作業時間のプレビュー** が表示されます。
- 終了時刻が開始以前の場合、**翌日の終了** として解釈します（例: 23:00 → 01:00 は翌日 1 時）。プレビューに「（終了は翌日）」と表示されます。

パネル右上の **作業 n分** バッジは、シフト − 休憩 − 停止 の **作業時間** を示します。

### 5.6 ⑥ 不良（項目別）

![不良入力](./images/InspectionActualRegistration/06_defects.png)

- 工程別不良項目マスタ（KT09）を **帰属工程** ごとにグループ表示します。
- 各項目：**−** / 数量入力 / **+** で数量を変更できます。
- 数量が 1 以上のカードは **ハイライト** 表示されます。
- ラベル横に **不良合計** バッジが表示されます。

### 5.7 ⑦ 備考・保存・クリア

| ボタン | 説明 |
|--------|------|
| **備考** | 任意、最大 500 文字（`manual_registration_note` に保存） |
| **保存** | 新規登録または更新（[§7](#7-新規登録の標準フロー)） |
| **クリア** | 編集状態を解除し、フォームを空にする（生産日は維持） |

保存成功後、検査員だけ残してフォームがリセットされ、一覧が再読込されます。

## 6. 登録一覧

![登録一覧](./images/InspectionActualRegistration/07_list_table.png)

### 6.1 ヘッダー操作

| 要素 | 説明 |
|------|------|
| 生産日・件数 | 現在表示中の生産日と行数 |
| **検査員** フィルタ | 特定検査員の行のみ表示（フォームの生産日とは独立） |
| **更新** | サーバーから一覧を再取得 |

### 6.2 テーブル列

| 列 | 内容 |
|----|------|
| 生産日 | `production_day` |
| 検査員 | 氏名（フルネーム優先） |
| CD / 製品名 | 製品コード・名称 |
| 生産数 | 本数 |
| 不良 | 不良合計 |
| 能率 | 本/時（整数）。200 未満または 800 超は **赤** 表示 |
| 開始 / 終了 | 日時（月/日 時:分） |
| 休憩 / 停止 | 分表示（0 は `—`） |
| 取得元 | MES / Excel / CSV（[§13](#13-取得元と編集制限)） |
| 備考 | 手動登録備考 |
| 操作 | 編集・削除（権限・状態による） |

### 6.3 行の操作

| 操作 | 条件 |
|------|------|
| **行クリック** | 編集可能な行のみフォームに読込 |
| **編集アイコン** | `canEdit` かつ MES 生産中でない行 |
| **削除アイコン** | `canDelete` 権限がある場合 |
| **—（ロック）** | MES 生産中で編集不可。ツールチップ「MES生産中」 |

編集中の行はテーブル上 **ハイライト** され、フォーム見出しが **「編集中 #ID」** になります。

## 7. 新規登録の標準フロー

![標準フロー](./images/InspectionActualRegistration/08_workflow.png)

1. **① 生産日** を確認・設定する（必要なら前日/翌日/今日ボタン）。
2. **② 検査員** を選択する。
3. **③ 製品名** を選択する。
4. **④ 生産数** を入力する（箱数または本数）。
5. **⑤ 生産時間** … 開始・終了・休憩・停止を入力する。
6. **⑥ 不良** … 該当項目があれば数量を入力する。
7. **⑦ 備考** … 必要ならメモを入力する。
8. **保存** をクリックする。

**サーバー処理（参考）**

1. `inspection_management` に行を新規作成（`manual_registration: true`）。
2. 続けて PATCH で生産数・時刻・不良・完了フラグ等を一括更新します。
3. `production_completed_check: true` となり、**確定済み実績** として扱われます。

## 8. 編集モード

![編集モード](./images/InspectionActualRegistration/09_edit_mode.png)

| 項目 | 編集時の挙動 |
|------|--------------|
| パネル見出し | `編集中 #123` ＋ **編集** タグ（オレンジ枠） |
| 製品名 | 変更不可（読み取り専用） |
| 検査員 | 変更可 |
| 生産日 | 変更可（保存時に行の生産日も更新） |
| その他 | 新規登録と同様 |

**編集できない行**

- **MES 生産中**（開始時刻あり・終了時刻なし）の行は手動編集不可です。  
  → 先に **[検査実績収集](./inspection-actual_ja.md)** 画面で生産終了するか、該当端末での操作を完了してください。

## 9. 削除

![削除確認](./images/InspectionActualRegistration/10_delete_confirm.png)

1. 一覧の **削除** アイコンをクリックします。
2. 確認ダイアログに検査員・製品名が表示されます。
3. 次の場合は **追加警告** が表示されます。
   - **MES 生産中** のデータ
   - **Excel 取込** データ
   - **CSV 取込** データ
4. **削除** で確定すると、行がサーバーから削除されます。

編集中の行を削除した場合、フォームも自動的にクリアされます。

## 10. 生産数（箱数・本数）のルール

| ルール | 説明 |
|--------|------|
| 保存必須 | **本数** が 0 以上の整数であること |
| 入数あり・箱数入力 | 箱数から本数を自動計算 |
| 入数なし | 箱数入力不可。**本数のみ** 入力 |
| 割り切れない本数 | 警告表示。保存時「入力した本数のまま登録しますか？」と確認 |

**例（入数 50）**

- 箱数 3 → 本数 150（自動）
- 本数 155 → 箱数 3（四捨五入表示）、保存時に確認ダイアログ

## 11. 生産時間・休憩・停止のルール

| ルール | 説明 |
|--------|------|
| 開始・終了必須 | 両方有効な時刻であること |
| 休憩＋停止 | **シフト時間（終了−開始）を超えられない** |
| 翌日跨ぎ | 終了 ≤ 開始 のとき、終了は **翌日** として計算 |
| 作業時間 | シフト − 休憩 − 停止（分）。パネルバッジ・プレビューに表示 |
| サーバー保存 | `mes_break_sec` / `mes_stop_sec` / `mes_paused_accum_sec` / `mes_net_production_sec` を算出して保存 |

**用語の整理（本画面）**

| 用語 | 意味 |
|------|------|
| **休憩** | 計画的な休憩（分） |
| **停止** | 段替・待ち等（分） |
| **シフト** | 開始から終了までの経過時間 |
| **作業** | シフト − 休憩 − 停止 |

※ [検査実績収集](./inspection-actual_ja.md) の「一時停止」はタイマー操作ですが、本画面では **停止（分）** として事後入力します。

| 操作 | 説明 |
|------|------|
| **+** | 数量 +1 |
| **−** | 数量 −1（0 で項目削除） |
| 数量直接入力 | 数値のみ。0 以下は項目から除外 |

不良項目がマスタ未登録の場合、「不良項目なし」と表示されます。マスタ整備後、画面を再読込してください。

## 13. 取得元と編集制限

一覧の **取得元** 列は、データの由来を示します。

| 表示 | 意味 | タグ色 |
|------|------|--------|
| **MES** | MES 実績収集または手動登録 | 緑 |
| **Excel** | Excel 同期取込 | 橙 |
| **CSV** | CSV 取込 | 灰 |

| 取得元 | 編集 | 削除 |
|--------|------|------|
| MES（確定済み） | ○（権限あり・生産中でない） | ○（`canDelete`） |
| MES（生産中） | × | △（警告付きで可能） |
| Excel / CSV | ○（権限あり・生産中でない） | △（警告付きで可能） |

手動登録で保存した行も、取得元は **MES** と表示されます（`manual_registration: true`）。

## 14. 検査実績収集（MES）との違い

| 項目 | 検査実績収集登録（本画面） | 検査実績収集 |
|------|---------------------------|--------------|
| 用途 | 事後入力・修正・削除 | 現場リアルタイム計測 |
| 検査員 | プルダウンで任意選択 | ログインユーザー固定 |
| タイマー | なし（時刻を直接入力） | 開始/停止/休憩ボタン |
| オフライン | 非対応（オンライン必須） | 対応 |
| 端末ロック | なし | あり |
| 製品読取 | なし | バーコード/QR |
| 完了状態 | 保存時に即確定 | 生産終了ダイアログで確定 |

## 15. 権限と操作制限

画面の操作は **MES モジュールの操作権限** に依存します。

| 権限 | 操作 |
|------|------|
| `canCreate` | 新規 **保存** |
| `canEdit` | **保存**（編集時）、一覧からの編集読込 |
| `canDelete` | 一覧の **削除** |

権限がない操作を試みると、システム共通の権限ガードで阻止されます。

## 16. 事前マスタ・環境

| 種別 | 要件 |
|------|------|
| 製品マスタ | 検査用 CD（末尾 `1`）、**入数** 設定推奨 |
| ユーザー | 検査員候補は **仕上課** 所属・有効ユーザー |
| 組織マスタ | 「仕上課」セクションが登録されていること |
| 工程別不良項目 | `detection_process_cd = KT09` |
| メニュー権限 | `MES_ACTUAL_REG_INSPECTION` |
| ブラウザ | Chrome / Edge 最新版推奨 |
| ネットワーク | **常時オンライン**（オフラインキューなし） |

---

## 17. 指標の計算方法

### 17.1 能率（一覧列）

```
能率 = 生産数 ÷ 作業時間[時間]
```

- 作業時間（秒）= 壁時計（開始〜終了）− 一時停止累計（休憩＋停止）
- 結果は **整数** に四捨五入（本/時）
- **200 未満** または **800 超** は赤色表示（入力ミス・異常値の目安）

### 17.2 不良（一覧列）

- 行の `defect_qty`（項目別不良の合計）を表示します。

## 19. 運用上の注意

1. **リアルタイム記録との使い分け**  
   現場タブレットでの計測は **検査実績収集**、紙からの転記・事後修正は **本画面** を使い分けてください。

2. **MES 生産中データ**  
   検査実績収集で計測中の行は本画面から編集できません。二重入力を避けるため、生産中は MES 画面で完結させてください。

3. **検査員の選択**  
   ログインユーザーと検査員が異なっても登録できます（事務代行入力を想定）。誤選択に注意してください。

4. **生産日の一致**  
   フォームの生産日と実際の作業日が一致しているか、保存前に必ず確認してください。

5. **Excel/CSV 取込データの削除**  
   削除確認に警告が出ます。取込元データとの整合を確認してから削除してください。

6. **能率の赤表示**  
   時刻・生産数・休憩/停止の入力ミスで異常値になることがあります。保存前にプレビューで作業時間を確認してください。
