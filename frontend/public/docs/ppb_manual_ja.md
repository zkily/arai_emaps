# 生産計画ベースライン管理 画面 操作説明

対象画面：`erp/production/planning/ProductionPlanBaselineManagement.vue`

本ページ内容は「生産計画ベースライン管理」画面のユーザー向け操作手順をまとめたものです。  
月次のベースライン計画を生成・修正し、現行計画・実績との比較（推移/差異）と、工程別PDF報告書の発行までを説明します。

---

## 1. 画面構成（概要）

- 画面上部：再取得（比較データの再読込）
- 上部カード（合并的操作区域）
  - ベースライン生成：基準月・工程を指定し、ベースラインを生成／削除／修正（基準計画の編集）
  - 比較条件：基準月・工程を指定し、ベースライン比較一覧を検索／クリア
- 本体：比較一覧（工程別タブ）＋操業度（別テーブル）
- ダイアログ
  - ベースライン計画修正：基準月の計画値を表形式で再調整（行ごとの保存/削除、変更を一括保存）
  - 基準計画の入力（固定化入力）：対象工程（メッキ/検査）で固定入力を行い、生成時に反映
  - 工程別PDF発行（進捗）：PDF生成・保存中の進捗表示

## 3. 操作手順

### 3.1 ベースライン生成（固定化して比較基準を作成）

1. 画面上部のカード「ベースライン生成」で `基準月` を選択します。
2. `工程` を選択します（「全工程」または個別工程）。
3. `生成` ボタンをクリックします。
4. 生成が完了すると、比較データが更新されます（画面内の比較一覧が最新状態になります）。
5. 生成対象が `メッキ` または `検査` の場合、`基準計画の入力` ダイアログが表示されます（平日・土曜・日曜の固定入力が必要です）。

 - ![ベースライン生成エリア](./images/ProductionPlanBaselineManagement/baseline_generate_area.png)

---

### 3.2 ベースライン削除（対象月/工程を削除）

1. 「ベースライン生成」で `基準月` を確認します。
2. `工程` を必要に応じて指定します（指定あり：その工程のみ、指定なし：全工程）。
3. `削除` ボタンをクリックします。
4. 確認ダイアログで `削除` を押すと、対象のベースラインが削除されます。
5. 削除後、比較一覧が更新されます。

---

### 3.3 比較条件で一覧を検索（ベースライン比較一覧）

1. 下部カード「比較条件」で `対象月` を選択します。
2. `工程` を必要に応じて選択します（「全工程」または個別）。
3. `検索` ボタンをクリックします。
4. 比較一覧（「ベースライン比較一覧」）が更新され、工程別タブが表示されます。
5. `クリア` を押すと、検索条件がリセットされます（再度 `検索` が必要です）。

- ![比較条件エリア](./images/ProductionPlanBaselineManagement/comparison_filter_area.png)

---

### 3.4 ベースライン比較一覧（工程別タブで差異を確認）

比較一覧は、工程ごとにタブ表示されます。タブを切り替えると、その工程の月別データ行が表示されます。

主な表列（工程タブ内）：

1. `日付`：`plan_date`（日付）
2. `基準計画`：ベースライン生成時に固定化された計画値（`baseline_plan`）
3. `現行計画`：現行計画（`production_plan_updates` を優先し、無い工程は `production_summarys` の plan を日次合計）
4. `計画差異`：`現行計画 - 基準計画`
5. `現行実績合計`：当月の現行実績を再集計した合計（`stock_transaction_logs`）
6. `計画対実績差`：ベースライン計画 - 現行実績

また、工程タブ内には `合計` ブロックが表示されます（現行計画合計・計画差異合計・現行実績合計・計画対実績差）。

- ![ベースライン比較一覧](./images/ProductionPlanBaselineManagement/comparison_list.png)

---

### 3.5 ベースライン計画修正（ダイアログで編集・保存・削除）

`計画を修正` ボタンから、ベースライン計画の行データを編集します。

1. 上部カードの `計画を修正` ボタンをクリックします。
2. `ベースライン計画修正` ダイアログが開きます。
3. （必要に応じて）`基準月` と `工程` を確認し、`データ取得` をクリックします。
4. 表の `基準計画` 欄にある数値を入力し、`保存` をクリックします（行ごとの即時保存）。
5. 行の `削除` をクリックすると、確認ダイアログが表示されます。削除が完了すると該当行が消え、比較一覧も更新されます。
6. ダイアログ下部の `変更を一括保存` を押すと、変更があった行だけまとめて保存されます。
7. `閉じる` でダイアログを閉じます。

- ![ベースライン計画修正ダイアログ](./images/ProductionPlanBaselineManagement/adjustment_dialog.png)

---

### 3.6 固定化入力（メッキ/検査向け：平日一律＋土日任意）

`メッキ` または `検査` のベースライン生成時は、固定化入力が必要です。

1. `基準計画の入力` ダイアログで `工程` が対象工程として表示されます。
2. `平日（月〜金）の基準計画数（必須）` に値を入力します（必須、1以上）。
3. `土曜（任意）` と `日曜（任意）` は必要に応じて入力します（未入力の場合は週末行を作りません）。
4. `生成` をクリックすると、入力内容がベースライン生成に反映されます。
5. `キャンセル` でダイアログを閉じることもできます。

---

### 3.7 操業度（production_plan_rate）の確認

「操業度」カードでは、指定した `対象月` と `工程` に対して、操業度差異の一覧を表示します。

1. 操業度カードで `対象月` を選択します。
2. `工程` を選択します（必要に応じて「全て」）。
3. `検索` をクリックします。
4. テーブルが更新されます。

表列（操業度テーブル）：

1. `月`
2. `工程`
3. `ラインCD`
4. `ライン`
5. `操業度差異`（表示上はマイナス/プラスで色分けされた差異テキスト）
6. `ファイル名`
7. `取込日時`

- ![操業度カード](./images/ProductionPlanBaselineManagement/operation_rate_card.png)

---

### 3.8 工程別報告書発行（PDFの生成・保存）

画面右上の `工程別報告書発行` ボタンから、工程別のPDFを生成し、サーバーフォルダへ保存します。

1. 比較一覧が表示されている状態で、`工程別報告書発行` をクリックします。
2. `工程別PDF発行` の進捗ダイアログが表示され、PDF生成・保存が進行します。
3. 完了すると、正常保存結果が通知されます（保存件数やエラーメッセージ）。

---

## 4. 注意点（運用上のコツ）

1. `ベースライン生成` は対象データが「上書き」されます。再生成前に意図した変更か確認してください。
2. `計画を修正` は、表の数値編集後に `保存`（行ごと）または `変更を一括保存`（まとめて）で反映されます。
3. `削除` は確認ダイアログが出ます。削除後は比較一覧も更新されるため、影響範囲を把握してから実行してください。
4. PDF発行は比較データが必要です。データがない場合は `検索` を先に実行してください。
