## 1. 業務の目的と概要

### 1.1 ベースラインとは

**ベースライン** は、ある **基準月** の生産計画を固定したスナップショットです。月初または計画確定時点で生成し、以降の **現行計画** の変更や **実績** の進捗と比較して、計画のぶれや達成状況を可視化します。

### 1.2 典型的な運用サイクル

| 段階 | 作業 | 画面上の操作 |
|------|------|----------------|
| 月初 | 対象月のベースラインを生成 | **ベースライン生成** → **生成** |
| 月中 | 計画変更・実績の差異を確認 | **比較条件** → **検索**、工程タブを切替 |
| 必要時 | 基準計画の手修正 | **計画を修正** ダイアログ |
| 報告 | 工程別 PDF を出力 | **工程別報告書発行** |
| 設備負荷 | 操業度・差異工時を確認 | **操業度** カード（集計月・工程） |

## 2. 画面へのアクセス

1. Smart-EMAPs にログインします。
2. 左サイドメニューから次の順に開きます。  
   **ERP管理メニュー** → **生産管理** → **生産計画** → **計画ベースライン**
3. 画面タイトル **「生産計画ベースライン管理」** と説明文 **「月次計画を固定化し、現行計画・実績と比較して推移を把握します」** が表示されます。
4. タイトル横の **?** アイコンから、本マニュアル（ヘルプ）を別タブで開けます。

## 3. 画面構成

画面は **上から下** に次のブロックで構成されています。

![画面全体](./images/ProductionPlanBaselineManagement/overview.png)

| ブロック | 位置 | 内容 |
|----------|------|------|
| **ヘッダー** | 最上部 | 画面タイトル、ヘルプ、**再取得**（比較データの再読込） |
| **操作カード** | 上部 | **ベースライン生成** と **比較条件**（同一カード内の左右） |
| **サマリーカード** | 操作カード下 | 月次 KPI（基準計画合計・計画差異・達成率など） |
| **比較一覧** | 中央 | 工程別タブ＋日次比較表、**工程別報告書発行**／**印刷** |
| **操業度** | 下部 | APS 設備操業度（成型計画一覧と同一ロジック） |

### 3.1 ベースライン生成エリア

![ベースライン生成エリア](./images/ProductionPlanBaselineManagement/baseline_generate_area.png)

| 項目 | 説明 |
|------|------|
| **基準月** | ベースラインを作成・削除・修正する対象月（月ピッカー） |
| **工程** | **全工程** または個別工程（切断／面取／成型／メッキ／溶接／検査／外注メッキ／外注溶接） |
| **生成** | 選択月・工程のベースラインを生成（既存データは上書き） |
| **削除** | 選択月・工程のベースラインを削除 |
| **計画を修正** | ベースライン計画修正ダイアログを開く |

> **補足**：成型・溶接は `production_summarys` の `molding_plan`／`welding_plan` を基準計画に反映します（画面説明文参照）。

### 3.2 比較条件エリア

![比較条件エリア](./images/ProductionPlanBaselineManagement/comparison_filter_area.png)

| 項目 | 説明 |
|------|------|
| **対象月** | 比較一覧・サマリーを表示する月（ベースライン生成と **同一の月ピッカー** を共有） |
| **工程** | 絞り込み用。未選択時は全工程 |
| **検索** | 比較データを API から再取得 |
| **クリア** | 基準月を当月に戻し、工程をクリアして再検索 |

## 4. サマリーカード（KPI）

**検索** または **再取得** の後、操作カード直下に **7 枚** のサマリーカードが表示されます。

| カード名 | 意味 | 算出の目安 |
|----------|------|------------|
| **基準計画合計** | ベースライン計画の月合計 | 生成時に固定した計画値の合計 |
| **現行計画合計** | 最新の現行計画の月合計 | 工程ごとの現行計画ロジックに基づく日次合計 |
| **計画差異** | 現行計画 − 基準計画 | プラスは増加、マイナスは減少（色分け表示） |
| **現行実績合計** | 当月実績の再集計合計 | `stock_transaction_logs` から集計 |
| **計画対実績差** | 基準計画 − 現行実績 | ベースラインに対する実績の乖離 |
| **計画達成率** | 現行実績 ÷ 現行計画 | 現行計画が 0 の場合は表示なし |
| **達成率差異** | 計画対実績差 ÷ 基準計画合計 | 基準計画が 0 の場合は表示なし |

データが無い場合は各カードに `-` が表示されます。

## 5. ベースライン生成

### 5.1 手順

1. **ベースライン生成** エリアで **基準月** を選択します。
2. **工程** を選択します（**全工程** または個別）。
3. **生成** をクリックします。
4. 確認ダイアログ **「対象月のベースラインを再生成します。既存データは上書きされますがよろしいですか？」** で **生成** を押します。
5. 完了メッセージ **「ベースラインを生成しました」** の後、比較一覧・サマリーが自動更新されます。

### 5.2 工程別の生成ロジック（概要）

| 工程区分 | 基準計画の主なソース |
|----------|----------------------|
| **成型** | `production_summarys.molding_plan` の日次合計 |
| **溶接** | `production_summarys` の溶接計画（サマリベース） |
| **メッキ・検査** | [10 章](#10-基準計画の入力メッキ検査) の固定入力ダイアログ |
| **その他** | Excel 取込（`production_plan_updates`）優先、無い日はサマリ各 plan 列で補完 |

### 5.3 注意

- 再生成は **上書き** です。意図した月・工程か必ず確認してください。
- **メッキ** または **検査** を単独選択して生成した場合、先に **基準計画の入力** ダイアログが開きます（[10 章](#10-基準計画の入力メッキ検査)）。

---

## 6. ベースライン削除

1. **基準月** を確認します。
2. **工程** を指定します。  
   - **指定あり**：その工程のみ削除  
   - **全工程**（未選択）：対象月の **全工程** を削除
3. **削除** をクリックし、確認ダイアログで **削除** を押します。
4. **「ベースラインを削除しました」** 後、比較一覧が更新されます。

---

## 7. 比較条件と一覧の検索

1. **比較条件** で **対象月**（＝表示したい基準月）を選びます。
2. 必要に応じて **工程** で絞り込みます。
3. **検索** をクリックします。
4. サマリーカードと **ベースライン比較一覧** が更新されます。
5. **クリア** は、基準月を **当月** に戻し、工程フィルタを外したうえで再検索します。

画面上部の **再取得** も、現在の比較条件で一覧を再読込します（読込中はボタンがローディング表示になります）。

## 8. ベースライン比較一覧

![ベースライン比較一覧](./images/ProductionPlanBaselineManagement/comparison_list.png)

### 8.1 ヘッダー操作

| ボタン | 説明 |
|--------|------|
| **工程別報告書発行** | 工程別 PDF をサーバーへ保存（[11 章](#11-工程別報告書発行印刷)） |
| **印刷** | ブラウザ印刷ダイアログで比較表を印刷（データ 1 行以上で有効） |

ヘッダーには **対象年月** タグと **全 ○○ 行** の件数タグが表示されます。

### 8.2 工程別タブ

タブはデータがある工程のみ表示されます（例：切断、面取、成型、メッキ、溶接、検査、外注メッキ、外注溶接）。タブ名の横に **行数** が表示されます。タブを切り替えると、その工程の **日次** データが表に表示されます。

### 8.3 表の列

| 列名 | 内容 |
|------|------|
| **日付** | `plan_date`（YYYY-MM-DD） |
| **基準計画** | ベースライン生成時に固定した計画値 |
| **現行計画** | 最新の現行計画（工程により算出ロジックが異なる。列ヘッダの ? で詳細表示） |
| **計画差異** | 現行計画 − 基準計画（↑↓ アイコンと色分け） |
| **現行実績合計** | 当日までの実績再集計（データ無しは `-`） |
| **計画対実績差** | 基準計画 − 現行実績 |

> **現行計画の特例**：切断・面取・メッキ・検査・外注系は **現行計画＝基準計画**（常に同期）。成型は `molding_plan`、溶接は `welding_actual_plan` 等、工程ごとにツールチップの説明を参照してください。

### 8.4 タブ下部の「合計」

各工程タブの表の下に **合計** ブロックがあり、次を表示します。

- 現行計画合計
- 計画差異合計
- 現行実績合計
- 計画対実績差合計

## 9. ベースライン計画修正

![ベースライン計画修正ダイアログ](./images/ProductionPlanBaselineManagement/adjustment_dialog.png)

### 9.1 ダイアログを開く

1. 操作カードの **計画を修正** をクリックします。
2. **ベースライン計画修正** ダイアログが開き、画面上部の **基準月・工程** が初期値として入ります。
3. 自動で **データ取得** が走り、該当行が表に表示されます。

### 9.2 ツールバー

| 操作 | 説明 |
|------|------|
| **基準月** | 修正対象月 |
| **工程** | 絞り込み（全工程可） |
| **データ取得** | 条件に合うベースライン行を再読込 |
| **リセット** | 基準月を画面の値に戻し、工程をクリアして一覧を空にする |

### 9.3 行の編集

| 列 | 操作 |
|----|------|
| **日付** | 表示のみ |
| **工程** | タグ表示 |
| **基準計画** | **現在値** の下の数値入力欄を変更。矢印キー／Enter で次行へ移動可能 |
| **操作** | **保存**（行単位の即時反映）、**削除**（確認後に行削除し比較一覧も更新） |

### 9.4 一括保存と閉じる

- **変更を一括保存**：`現在値` と異なる行だけ順に保存します。変更が無い場合は **「変更された項目がありません」** と表示されます。
- **閉じる**：ダイアログを閉じます（未保存の入力は破棄されるため、必要なら先に保存してください）。

## 10. 基準計画の入力（メッキ・検査）

**メッキ** または **検査** を選択して **生成** した場合のみ、次のダイアログが表示されます。

![基準計画の入力ダイアログ](./images/ProductionPlanBaselineManagement/fixed_baseline_dialog.png)

| 項目 | 必須 | 説明 |
|------|------|------|
| **平日（月〜金）の基準計画数** | ○ | 1 以上の整数。平日各日に同じ値を書き込み |
| **土曜** | — | 任意。未入力の週末は行を作りません |
| **日曜** | — | 任意 |

1. 平日の値を入力します。
2. 必要なら土日を入力します。
3. **生成** で確定し、ベースライン生成 API が実行されます。
4. **キャンセル** でダイアログを閉じ、生成は行いません。

## 11. 工程別報告書発行・印刷

### 11.1 工程別報告書発行（PDF）

**前提**：比較一覧にデータがあること（`canExportBaselinePdf` が有効）。

1. 比較一覧ヘッダーの **工程別報告書発行** をクリックします。
2. 進捗ダイアログが表示され、工程ごとに PDF を生成・サーバーへ保存します。

![工程別PDF発行 進捗](./images/ProductionPlanBaselineManagement/export_pdf_progress.png)

3. 完了後、保存件数などの結果が通知されます。処理中はダイアログを閉じられません。

### 11.2 比較一覧の印刷

- **印刷** ボタンで、表示中の比較内容をブラウザ印刷します（一覧に 1 行以上必要）。

## 12. 操業度の確認

![操業度カード](./images/ProductionPlanBaselineManagement/operation_rate_card.png)

**操業度** は、成型計画一覧の **設備操業度** と同一データソース（`fetchSchedulingGrid` ＋ 設備別集計）です。ベースライン比較とは **独立した集計月・工程** で操作します。

### 12.1 操作

| 項目 | 説明 |
|------|------|
| **集計月** | `YYYY-MM` 形式（変更すると自動再読込） |
| **工程** | 工程 CD — 名称のプルダウン（マウント時に工程一覧を取得） |
| **印刷** | 操業度表のブラウザ印刷（データがある場合のみ有効） |

画面上部のチップ表示：

- **対象：○○年○○月**
- **操業度＝各時間÷理論稼働**
- **差異工時＝上記期間の Σ((実績−計画)/能率)**

### 12.2 表の列

| 列名 | 説明 |
|------|------|
| **設備** | ライン／設備名 |
| **指示数** | スケジュール件数 |
| **理論稼働(H)** | 理論稼働時間 |
| **計画数** / **実績数** | 数量 |
| **計画時間(H)** / **実績時間(H)** | 時間 |
| **計画操業度** / **実績操業度** | パーセント表示 |
| **操業度差異(H)** | 時間差（マイナスは色付き） |
| **差異操業度(%)** | 操業度の差異率 |

データが無い場合は **「該当データがありません。集計月・工程を変更すると自動で再読込されます。」** と表示されます。

## 13. よくあるメッセージと対処

| メッセージ | 原因 | 対処 |
|------------|------|------|
| 対象月を選択してください | 基準月未選択で生成／削除 | 月ピッカーで基準月を指定 |
| 基準月を選択してください | 修正ダイアログで月未設定 | 基準月を選択してデータ取得 |
| 該当データがありません | 条件に合うベースライン行なし | 先にベースラインを生成するか条件を見直し |
| 変更された項目がありません | 一括保存時、値に差分なし | 数値を変更してから一括保存 |
| 平日の基準計画数を入力してください（1以上の数） | メッキ／検査の固定入力不足 | 平日に 1 以上を入力 |
| 比較データの取得に失敗しました | API／ネットワークエラー | 再取得・権限・サーバー状態を確認 |
| ベースライン生成に失敗しました | 生成 API エラー | 対象月・工程・元データを確認 |
| 工程一覧の取得に失敗しました | 操業度用工程マスタ取得失敗 | 再読込、マスタデータを確認 |
