## 3. 画面構成と基本概念

画面は **上から下** に、次の 5 ブロックで構成されています。

| ブロック | 位置 | 内容 |
|----------|------|------|
| **① 生産ロット一覧** | 上部左（約 50%） | 計画ロット（instruction_plans）の一覧。切断指示の元データ |
| **② 製品情報・設備能率** | 上部右（約 50%） | 選択製品のマスタ情報と切断設備能率 |
| **③ 切断指示＋使用材料数** | 中部 | 切断指示-今日／翌日、使用材料数（材料別）- 今日／翌日 |
| **④ 面取ロット＋面取指示** | 中下部 | 面取ロット一覧、面取指示-今日／翌日、設備能率（面取） |
| **⑤ カンバン発行** | 最下部 | 現品票（カンバン）の待発行／発行済／完了管理 |

### 3.1 データの流れ（重要）

```
生産ロット一覧（instruction_plans）
    │ D&D または登録ダイアログ
    ▼
切断指示（cutting_management）─────► 面取工程あり製品の場合
    │                                    ├─► 面取ロット（chamfering_plans）自動生成
    │                                    ├─► 面取指示（chamfering_management）自動生成
    │                                    └─► カンバン（待発行）自動生成
    │
    ├─► 計画印刷 / 指示書発行
    ├─► 実績確定 → 在庫トランザクション
    └─► 使用材料数 → 材料在庫反映
```

- **生産ロット** を **切断指示** へ移すと、製品マスタに面取工程（KT02）がある場合は **面取ロット・面取指示・カンバン（待発行）** も連動して作成されます。
- 切断指示を **生産ロット一覧へ D&D で戻す** と、紐づく切断指示・面取指示・カンバンが削除され、ロットのみが残ります（[6 章](#6-ドラッグドロップ操作) 参照）。

### 3.2 「今日」と「翌日」の使い分け

| エリア | 用途 |
|--------|------|
| **切断指示-今日** | 当日の現場運用・実績入力・**実績確定**・計画印刷・指示書発行 |
| **切断指示-翌日** | 翌日（または指定日）の計画確認・ロットの事前配置 |
| **面取指示-今日／翌日** | 面取現場向けの当日／翌日計画 |
| **使用材料数-今日** | 当日の材料消費集計と **使用数反映** の対象 |
| **使用材料数-翌日** | 翌日分の材料使用見込み（参照用） |

> **推奨**：翌日分の指示書発行前に、**切断指示-今日** の日付を翌日に合わせるか、ロットを **切断指示-翌日** エリアへ事前配置してから **計画印刷**・**指示書発行** を行います。

## 6. ドラッグ＆ドロップ操作

![D&D：ロット→切断指示](./images/CuttingInstructionManual/06_drag_drop_lot_to_cutting.png)

本画面の核心操作の一つです。マウスで行をドラッグし、ドロップ先のゾーンが **ハイライト** された状態で離します。

### 6.1 生産ロット → 切断指示

1. **生産ロット一覧** の行をドラッグ開始
2. **切断指示-今日** または **切断指示-翌日** のテーブル領域へドロップ  
   （ヒント：**「ここにドロップでロットを切断指示へ移行」**）
3. **切断指示の登録** ダイアログが開く
4. **生産日** と **切断機** を指定し **登録**

![切断指示の登録ダイアログ](./images/CuttingInstructionManual/18_move_to_cutting_dialog.png)

| 項目 | 操作 |
|------|------|
| 生産日 | 日付ピッカー、前日／翌日ボタン、**今日** ボタン |
| 切断機 | 有効な切断機ボタンから選択（1 台） |

登録後：

- 切断指示一覧に行が追加される
- 面取工程がある製品は **面取ロット・面取指示・カンバン（待発行）** も自動作成
- 生産ロット一覧からは当該ロットが **消える**（切断指示に移行）

### 6.2 切断指示 → 生産ロット一覧（指示の取消）

1. **切断指示** の行をドラッグ
2. **生産ロット一覧** へドロップ  
   （ヒント：**「ここにドロップで切断指示をロットへ戻す」**）
3. 確認後、切断指示が削除されロットが復元。紐づく面取指示・カンバンも削除

### 6.3 切断指示内の並べ替え

- 切断指示の行を **別の行の上** にドロップすると **生産順** が入れ替わります
- テーブル先頭／末尾の **ドロップエッジ** にドロップすると、先頭／末尾への移動も可能

### 6.4 面取ロット → 面取指示

1. **面取ロット一覧** の行をドラッグ
2. **面取指示-今日** または **面取指示-翌日** へドロップ
3. **面取指示の登録** ダイアログで **生産日** と **面取機** を指定して登録

### 6.5 面取指示 → 面取ロット一覧

- 面取指示行を **面取ロット一覧** へ D&D  
  （ヒント：**「ここにドロップで面取指示をロットへ戻す」**）
- 面取指示が削除され、面取ロットに戻る

1. **切断指示-今日** の日付を対象日に設定
2. **計画印刷** をクリック
3. ブラウザ印刷ダイアログが開く（**A4 縦・余白小**）
4. 内容：切断機別計画、材料在庫、材料バラ在庫（サブ在庫で計画使用数未設定のもの）

### 7.6 指示書発行

1. 対象日の切断指示データがあることを確認
2. **指示書発行** をクリック
3. **切断機ごとに 1 ページ** の「**切断生産指示書**」が生成される（**A5 横向き**）
4. 各ページに CD・製品名・順位・原材料・生産数・備考、合計生産数
5. ブラウザ印刷ダイアログで印刷し、現場へ配布

> **重要**：原則 **翌日分** の指示書を前日夕方に発行する運用が一般的です。日付を翌日に合わせてから発行してください。

### 7.7 実績確定

1. 現場作業完了後、**生産数・不良・完了** 等を入力済みであることを確認
2. **切断指示-今日** の日付を **確定対象日** に設定
3. **実績確定** をクリック  
   ※ **現在選択中の切断機フィルタに関係なく、指定日の全切断機** が対象
4. 成功すると **実績確定結果** ダイアログ（登録件数・合計数量）が表示

![実績確定結果](./images/CuttingInstructionManual/20_confirm_actual_result.png)

## 8. 切断指示（翌日）

**切断指示-翌日** は、右側（画面幅 40%）に配置された **参照・事前配置** 用エリアです。

### 8.1 ヘッダー

- **日付**：`selectedDateTomorrow`（前日／翌日ボタン、日付ピッカー）
- 操作ボタン（計画印刷・指示書発行・実績確定）は **今日側のみ** にあります

### 8.2 一覧の列

CD・生産日・切断機・製品名・生産数・不良・生産順・生産時間（操作列なし）

### 8.3 使い方

1. 翌日のロットを **D&D** で翌日エリアへ事前配置
2. 生産日・生産順をダブルクリック編集で調整
3. 当日になったら日付を **今日** に合わせて **計画印刷**・**指示書発行** を実施

フッター：**生産数合計**、**不良合計**、**生産時間合計**、**使用材料数（新増）**

## 9. 使用材料数（材料別）

![使用材料数](./images/CuttingInstructionManual/10_usage_summary.png)

切断指示の直下に、**使用材料数（材料別）- 今日** と **- 翌日** の 2 テーブルがあります。

### 9.1 列の意味

| 列 | 説明 |
|----|------|
| 製品名 | 対象製品 |
| 原材料 | 材料名称 |
| 管理コード | 材料管理コード（空の場合ハイライト） |
| 在庫区分 | 主表／サブ（スイッチ） |
| 材料使用数 | 使用束数（ダブルクリックで編集） |
| 使用材料 | **反映済**／**未反映**／**サブ・手動** |

- **主表** 在庫：切断指示から自動集計。`使用数反映` 実行後 **反映済** になる
- **サブ** 在庫：**サブ・手動** 表示。使用数反映の対象外

### 9.2 使用数反映

1. **使用材料数（材料別）- 今日** の日付を確認
2. **使用数反映**（オレンジ色ボタン）をクリック
3. 確認ダイアログ：  
   「表示中の YYYY-MM-DD のデータを材料在庫に反映します。反映済の管理コードは反映しません。よろしいですか？」
4. **反映する** で実行 → 材料在庫（stock）を更新

> **タイミング**：切断 **実績確定** の後、または当日の切断作業完了後に実施する運用が一般的です。

### 9.3 指定日材料数

- **指定日材料数** ボタンで、任意日の材料使用一覧を印刷用ダイアログ表示
- 購買・倉庫担当との共有に利用

## 11. 面取指示（今日・翌日）

## 16. 標準的な日次業務フロー

### 16.1 前日夕方（翌日準備）

| 順 | 作業 | 操作場所 |
|:--:|------|----------|
| 1 | 翌日分ロットの有無を確認 | 生産ロット一覧 |
| 2 | 不足ロットを新規追加／データ管理で登録 | 生産ロット一覧 |
| 3 | ロットを **切断指示-翌日** へ D&D、生産日・切断機を指定 | 切断指示 |
| 4 | 面取が必要な製品は面取ロット／面取指示を確認 | 面取エリア |
| 5 | 日付を翌日に合わせ **計画印刷** | 切断指示-今日 |
| 6 | **指示書発行**（切断・面取） | 切断／面取指示-今日 |
| 7 | カンバン **一括発行**（待発行分） | カンバン発行 |

### 16.2 当日（現場＋事務）

| 順 | 作業 | 操作場所 |
|:--:|------|----------|
| 1 | 指示書・カンバンを現場へ配布 | 現場 |
| 2 | 作業中：生産数・不良・完了スイッチを更新 | 切断／面取指示-今日 |
| 3 | 未完了分は **翌日へ順延** | 切断／面取指示 |
| 4 | 作業完了後 **実績確定**（切断→面取の順が無難） | 切断／面取指示-今日 |
| 5 | **使用数反映** | 使用材料数-今日 |
| 6 | **切断済・面取済リスト** で漏れ確認 | ヘッダーボタン |

### 16.3 指示の取消・修正

- 誤登録：**切断指示 → 生産ロット** または **面取指示 → 面取ロット** へ D&D で戻す
- 数量修正：ダブルクリック編集
- カンバン再印刷：**再発行**

## 17. よくあるメッセージと対処

| メッセージ | 原因 | 対処 |
|------------|------|------|
| 生産日を選択してください | 日付未設定で印刷／発行／実績確定 | 日付ピッカーで対象日を設定 |
| 該当日のデータがありません | 指定日に切断指示が 0 件 | ロットを D&D するか新規追加 |
| 反映対象のデータがありません | 使用数反映時、対象行が 0 件 | 切断指示の有無・サブ在庫行を確認 |
| 実績確定に失敗しました | API エラー | ネットワーク／権限／入力値を確認。詳細は画面エラー表示 |
| 使用数反映に失敗しました | 材料在庫 API エラー | 管理コード・在庫マスタを確認 |
| 一覧で製品をクリック | 製品未選択 | ロット行を 1 回クリック |
| データなし | フィルタ条件に合致する行なし | フィルタ解除またはデータ登録 |
