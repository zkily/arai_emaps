| 項目 | 内容 |
|------|------|
| 対象アプリ | **SmartEMAP**（Android タブレット／端末） |
| メニュー | **MES管理メニュー** → **実績収集** → **検査実績収集** |

## 1. 画面の目的とできること

検査実績収集では、**1 製品 × 1 検査員（ログインユーザー）× 1 生産日** 単位で次を記録します。

| 機能 | 説明 |
|------|------|
| 生産タイマー | **生産開始** → **一時停止 / 休憩 / 再開** → **生産終了** の時刻と稼働時間を記録 |
| 不良入力 | 不良項目ごとに **＋ / −** で数量入力（生産中のみ） |
| 実績確定 | 終了時に **箱数** または **生産数（本数）** を入力してサーバーへ保存 |
| 当日履歴 | **本日の確定実績** を一覧表示・編集 |
| 製品読取 | ツールバーの **読取** でカメラを開き、バーコード / QR（5 桁数字）から検査製品を選択 |
| 生産中の箱 QR 読取 | **計測中**のみ画面右側中央に常駐カメラが起動。箱の QR（5 桁）を読み取り、**生産中製品と一致**したときのみサーバーへ記録。**一時停止／休憩中はカメラ停止** |
| 離席中表示 | **一時停止** または **休憩** 中、画面下部に大きく **「離席中」** を表示（再開／休憩再開で消える） |
| QR読取履歴 | ツールバー右端の **QR読取履歴** で、当該計画の読取一覧と合計箱数・本数を右側パネルに表示 |
| QR読取合計 | 生産終了ダイアログに、**当該計画（inspection_id）かつ生産開始〜終了**の読取回数／箱数／本数を表示 |
| オフライン | 通信切断中も計測継続。復帰後に自動アップロード |
| 次製品参考 | 検査モニタで指定された「次の製品」を表示し、**選ぶ** で検査製品に反映（反映後は次製品表示をクリア） |

**検査員について（重要）**

- 検査員は **ログイン中のユーザーに固定** されます（変更不可）。
- 別の検査員名義で記録する場合は、そのアカウントでログインし直してください。
- 確定実績の編集は、原則として **自分が検査員として記録した行のみ** 可能です。

## 2. 事前準備

| 項目 | 内容 |
|------|------|
| 端末 | 現場配付の Android タブレット（SmartEMAP インストール済み） |
| ネットワーク | 社内 Wi‑Fi 推奨。オフラインでも計測可能だが、確定後の同期には接続が必要 |
| アカウント | 検査員本人のユーザーでログイン |
| 製品マスタ | 検査用製品 CD が登録済みであること |
| 不良マスタ | 工程別不良項目（検査工程）が登録済みであること |
| 入数 | 製品の **入数（本/箱）** が設定されていると、終了時に箱数から本数を自動計算できます |
| カメラ権限 | 製品読取ダイアログおよび生産中の常駐カメラに必要 |

**対象製品の条件（アプリ側フィルタ）**

- 製品名に **「加工」「アーチ」** を含むものは一覧に出ません

## 3. 画面の開き方

1. SmartEMAP アプリを起動し、**検査員アカウントでログイン** します。

![ログイン画面（検査員アカウント）](./images/InspectionActualAndroid/03_login.png)

2. 左上（またはメニューアイコン）からサイドメニューを開きます。
3. **MES管理メニュー** → **実績収集** → **検査実績収集** をタップします。

![メニューから検査実績収集を開く](./images/InspectionActualAndroid/02_menu_navigation.png)

> **注意**  
> 同メニュー内の **「検査実績登録」** は手動一括登録用の別画面です。現場でのリアルタイム計測は必ず **「検査実績収集」** を使ってください。

ダッシュボードにショートカットがある場合は、「検査実績収集」タイルからも開けます。

## 6. ツールバー（生産日・検査員・製品・QR履歴）

ツールバーは 1 行です。左側に生産日・検査員・製品、**読取** の右隣〜行の最右端に **QR読取履歴** があります。

### 6.1 生産日

| 操作 | 説明 |
|------|------|
| 日付表示 | 対象の生産日（日本時間基準） |
| **前日** | 1 日前へ |
| **今日** | 当日へジャンプ |
| **翌日** | 1 日後へ |

生産日を変えると、その日の計画・確定実績・進行中セッションが再読込されます。QR読取履歴パネルを開いている場合は、条件に合わせて再取得されます。

### 6.2 検査員

| 項目 | 説明 |
|------|------|
| 表示 | ログイン中の氏名（またはユーザー名）が **読み取り専用** で表示 |
| 変更 | **不可** |

### 6.3 検査製品・読取

| 操作 | 説明 |
|------|------|
| プルダウン | 検査製品を選択 |
| **読取** | 製品選択用のカメラダイアログを開く（[§7.1](#71-製品選択用の読取ダイアログ)） |

**生産中は製品変更不可** です。先に生産終了してください。

製品未選択時は次の案内が表示されます。

> 「検査製品を選択すると作業を開始できます」

### 6.4 QR読取履歴ボタン

| 操作 | 説明 |
|------|------|
| 位置 | **読取** の右側、ツールバー行の **最右端** |
| タップ | 画面右側に **QR読取履歴** パネルを開く／再読込（[§7.3](#73-qr読取履歴パネル)） |
| 前提 | 検査製品を選択済みであること（未選択時は案内メッセージ） |

## 7. バーコード / QR 読取

アプリには次の読取関連機能があります。用途が異なるため混同しないでください。

| 種類 | いつ使うか | 目的 |
|------|------------|------|
| **読取**（ダイアログ） | 生産開始前（製品選択時） | 検査製品を選ぶ |
| **常駐カメラ** | **計測中のみ**（一時停止・休憩中は停止） | 箱の QR を読み、箱数実績を記録 |
| **QR読取履歴** | 製品選択後いつでも（生産中推奨） | 当該計画の読取一覧と合計を確認 |

### 7.1 製品選択用の読取（ダイアログ）

1. ツールバーの **読取** をタップします。
2. 「バーコード / QR 読取」ダイアログが開きます。
3. 枠内にコードを合わせます。
4. 読み取り成功後、検査製品が自動選択されます。

![バーコード / QR 読取](./images/InspectionActualAndroid/15_barcode_scan.png)

| 項目 | 説明 |
|------|------|
| 読取ルール | **5 桁の数字** のみ有効 |
| 照合 | 製品 CD の末尾が「`{5桁}1`」または「`{5桁}`」の検査製品と照合（`…1` を優先） |
| カメラ | **前面カメラ** / **背面カメラ** を切替可能 |
| 失敗時 | 「読取り再試行」や端末のカメラ権限を確認 |

### 7.2 生産中の常駐カメラ QR 読取（箱）

**生産開始**（位相が **計測中**）すると、画面 **右側の中央** に小さなカメラプレビュー（「QR読取（稼働中）」）が表示されます。  
箱に貼られた QR を連続で読み取れます。

| 項目 | 説明 |
|------|------|
| 表示条件 | **計測中** かつ本端末で操作中のときのみ表示 |
| 非表示になるとき | **一時停止**／**休憩**（カメラ機能も停止）／**生産終了**／セッション終了 |
| 再表示 | **再開**／**休憩再開** で再び計測中になると、カメラが再起動する |
| 表示位置 | 画面右側・縦方向の中央付近（履歴パネル表示中は左へずれる場合あり） |
| カメラ | 既定は **前面カメラ**。プレビュー上の切替で背面にも変更可 |
| 読取ルール | **5 桁の数字** のみ処理（それ以外は無視） |
| 製品照合 | 読取コードから解決した **製品 CD** が、**いま生産中の検査製品 CD** と一致すること |
| 一致時 | 画面上部に **一行** で成功表示（例：`QR読取成功　製品CD · 製品名　時刻`）。サーバー表 `inspection_mes_qr_scan` に 1 件登録（`inspection_id` 付き） |
| 不一致時 | 「製品不一致」を表示。**登録しない** |
| 1 回あたりの記録 | 箱数 **1**、本数 ＝ **入数 × 1**（入数が設定されている場合） |
| 成功後の間隔 | 成功登録のあと、原則 **60 秒** たたないと次の成功登録はできません（「読取間隔」表示） |
| 二重読取防止 | 一度読んだあと、QR を枠から外すまで再発火しません。また **2 秒以内** の連続読取は無視します（移開時の誤検出対策） |

> **一時停止・休憩中は読取できません。**  
> カメラは閉じられ、万が一コールバックが来てもアプリ側で無視します。箱 QR を読む前に **再開**／**休憩再開** してください。

**作業のコツ**

1. 枠内に箱の QR を合わせ、成功表示を確認してから次の箱へ進みます。
2. 別製品の QR を誤って当てた場合は「製品不一致」となり、数量には入りません。
3. カメラ権限を求められた場合は許可してください。権限がないと常駐カメラは動作しません。
4. 離席（一時停止／休憩）中はカメラが止まるため、席に戻ったら必ず再開してから読取を続けてください。
5. 読取件数の確認は **QR読取履歴** パネルを使います。

### 7.3 QR読取履歴パネル

1. ツールバー右端の **QR読取履歴** をタップします。
2. 画面 **右側** にパネルが開きます。
3. 一覧・合計を確認します。更新アイコンで再取得、**閉じる（×）** でパネルを閉じます。

| 表示 | 説明 |
|------|------|
| 製品 | 現在選択中の製品 CD・名称 |
| **合計箱数** / **合計本数** | 下記の絞り込み条件での合計 |
| 回数 | 読取件数（×N） |
| 一覧列 | 時刻 / QR / 箱 / 本（新しい順） |

**絞り込み条件（重要）**

- **inspection_id**（いま操作中の検査計画）に紐づく読取のみ
- **生産開始時刻〜生産終了時刻** の間に登録された読取のみ（未終了の場合は現在時刻まで）

> 生産開始前や、計画 ID がまだサーバーに無い状態では、一覧が空になることがあります。

新しく箱 QR を読取・登録した直後、パネルが開いていれば内容は自動で更新されます。

## 8. 次の製品（参考）

担当者が Web **検査モニタ** で「次の製品」を指定している場合、画面に **次の製品（参考）** が表示されます。

| 操作 | 説明 |
|------|------|
| 表示 | 製品 CD・製品名（参考情報） |
| **選ぶ** | 検査製品にその製品を反映（生産中・ロック時は無効）。反映後、**次の製品（参考）の表示はすぐクリア** され、サーバ上の次製品指定も解除されます |

- 自動で生産開始はしません。必ず内容を確認してから **選ぶ**／手動選択してください。
- プルダウンで同じ製品を選んだ場合も、製品 CD／名称が一致していれば同様にクリアされます。
- 詳細な指定方法は **[検査モニタ 操作説明](./inspection-monitor_ja.md)** を参照。

## 9. 生産開始とタイマー操作

製品を選ぶと中央に **生産カード** が表示されます。

### 9.1 生産開始

1. 生産日・製品を確認します。
2. **生産開始** をタップします。
3. タイマーが **計測中** になり、稼働時間がカウントされます。
4. 同時に画面右側へ **常駐カメラ（QR読取）** が起動します（[§7.2](#72-生産中の常駐カメラqr読取箱)）。

![計測中の画面](./images/InspectionActualAndroid/20_workflow.png)

| 表示 | 説明 |
|------|------|
| **稼働時間** | 正味の計測時間（一時停止・休憩中は増え方に注意） |
| 位相ラベル | 未開始 / 計測中 / 一時停止中 / 休憩中 / 終了済 |
| 副指標 | 生産開始時刻・一時停止累計・休憩累計 |
| 常駐カメラ | **計測中**のみ表示。箱 QR の読取用 |
| 離席中 | **一時停止中／休憩中**に画面下部へ大きく表示 |

### 9.2 操作ボタン一覧

| ボタン | 使えるとき | 動作 |
|--------|------------|------|
| **生産開始** | 未開始 | 計測開始。常駐カメラ起動 |
| **一時停止** | 計測中 | 稼働カウントを一時停止。**カメラ停止**＋下部に **離席中** |
| **再開** | 一時停止中 | 計測再開。**カメラ再起動**、離席中表示を消去 |
| **休憩** | 計測中 | 休憩カウント開始。**カメラ停止**＋下部に **離席中** |
| **休憩再開** | 休憩中 | 計測へ戻る。**カメラ再起動**、離席中表示を消去 |
| **生産終了** | 計測中（一時停止・休憩中は不可の場合あり） | 終了ダイアログを開く |

## 10. 一時停止と休憩の違い

| 区分 | ボタン | 使いどころ | タイマーへの影響 | カメラ・画面表示 |
|------|--------|------------|------------------|------------------|
| 一時停止 | **一時停止** → **再開** | 短時間の中断、確認待ちなど | 稼働時間は増えず、一時停止累計が増える | カメラ停止／再開で再起動。下部に **離席中** |
| 休憩 | **休憩** → **休憩再開** | 休憩時間 | 稼働時間は増えず、休憩累計が増える | 同上（休憩中も **離席中**） |

![一時停止中](./images/InspectionActualAndroid/09_production_paused.png)

### 10.1 離席中の表示

**一時停止** または **休憩** をタップすると、画面 **下部中央** に大きな文字で **「離席中」** が表示されます（言語切替時は対応文言）。

| 項目 | 説明 |
|------|------|
| 表示タイミング | 一時停止中・休憩中（本端末で操作中） |
| 表示内容 | **「離席中」**（大きい文字）。軽い点滅アニメーション付き |
| 消えるタイミング | **再開**／**休憩再開**、または生産終了・セッション終了 |
| 目的 | 席を離れている／計測が止まっていることを周囲にも分かりやすくする |

### 10.2 カメラとの連動

| 操作 | 常駐カメラ | 離席中表示 |
|------|------------|------------|
| **一時停止** / **休憩** | **閉じる（停止）** | **表示** |
| **再開** / **休憩再開** | **再起動** | **消去** |
| **生産終了** | 閉じる | 消去 |

**注意**

- 一時停止中・休憩中は、状況により **生産終了が無効** になります。先に **再開** / **休憩再開** してから終了してください。
- 一時停止が長い場合、再開ボタンが強調表示されることがあります。
- 離席中は箱 QR を読まないでください。席に戻ったら再開してから読取を続けます。

## 11. 不良（項目別）の入力

| 項目 | 説明 |
|------|------|
| 見出し | **不良（項目別）** |
| ヒント | 「生産開始後に +/- で入力」 |
| グループ | **帰属工程** ごとに色分けされたブロック |
| 操作 | 各不良項目の **−** / **＋** |

![計測中の不良入力](./images/InspectionActualAndroid/08_production_running.png)

**ルール**

1. **生産開始後（本端末で操作中）のみ** 増減できます。
2. 数量は 0 未満になりません（− で 0 止まり）。
3. 不良合計は生産カード上部にも表示されます。
4. 不良項目が未登録の場合は「不良項目が未登録です」と表示されます（マスタ担当へ連絡）。

入力内容は随時サーバーへ同期されます（オフライン時は端末に保持）。

## 12. 生産終了・実績確定

1. 計測中に **生産終了** をタップします。
2. 「生産終了・実績確定」ダイアログが開きます。
3. 内容を確認し、数量を入力します。
4. **実績を確定** をタップします。

![生産終了ダイアログ](./images/InspectionActualAndroid/11_end_dialog.png)

### 12.1 ダイアログの項目

| 項目 | 説明 |
|------|------|
| 製品 CD / 製品名 | 確認用（編集不可） |
| 検査員 | ログインユーザー |
| 生産開始 / 生産終了 | 壁時計時刻 |
| 稼働時間 | 正味稼働 |
| 不良合計 | 終了時点の合計（0 より大きいとき表示） |
| **QR読取合計（本製品）** | 下記条件で集計した **読取回数 / 箱数 / 本数**（実績があるとき表示）。未入力の箱数・本数の参考値にも使えます |
| **箱数** | 入数設定ありのとき主入力 |
| **生産数** | 入数あり：箱数 × 入数で自動計算／入数なし：本数を直接入力 |

> **QR読取合計について**  
> - 集計対象は **当該検査計画（inspection_id）** かつ **生産開始〜生産終了** に登録された QR のみです（他製品・他セッション・時間外の読取は含みません）。  
> - 右側の **QR読取履歴** パネルと同じ条件です。  
> - 表示は参考です。最終の確定数量は、現場の実箱数に合わせて **箱数／生産数** を入力・確認してください。

### 12.2 入数がある場合

- 画面に「入数 ○○ 本/箱」と表示されます。
- **箱数** を入れると **生産数** が自動計算されます。
- 本数が入数で割り切れない場合は確認ダイアログが出ます。意図どおりなら **確定する** で続行できます。

### 12.3 入数がない場合

- 「入数未設定（本数を直接入力）」と表示されます。
- **生産数（本数）** を直接入力して確定します。

確定後はタイマーが終了し、下部の **本日の確定実績** に行が追加されます。常駐カメラは閉じます。

## 13. 本日の確定実績と修正

画面下部に、当日・ログイン検査員の確定済み実績が一覧表示されます。

![本日の確定実績](./images/InspectionActualAndroid/12_history.png)

| 列の例 | 説明 |
|--------|------|
| 製品 CD / 製品名 | 対象製品 |
| 箱数 | 入数から換算した箱数（表示） |
| 生産数 | 確定本数 |
| 不良数 / 不良率 | 不良合計と比率 |
| 能率 | 時間あたり生産の指標 |
| 生産開始 / 終了 | 時刻 |
| 稼働時間・一時停止 | 分単位などの表示 |
| 検査員 | 担当者 |
| **編集** | 確定実績の修正ダイアログ |

### 13.1 確定実績の修正

1. 行の **編集** をタップします。
2. 生産数・不良・開始/終了時刻・一時停止秒などを修正します。
3. **保存** でサーバーへ反映します（オンライン推奨）。

> ヒント（アプリ表示）：一時停止累計は秒単位。稼働は開始・終了・停止から自動計算されます。

**他の検査員の実績は編集できません。**

## 14. 検査生産中・セッション復旧

未終了の検査が残っている場合、ヘッダーの一覧アイコンや復旧バナーから操作できます。

| 表示 | 対処 |
|------|------|
| 「未終了の検査生産があります」 | **作業を再開** でタイマーと操作を本端末に接続 |
| 「検査生産中：○○」＋別製品選択中 | **作業を再開** するか、先に生産終了 |

**手順（再開）**

1. **作業を再開** をタップします。
2. 製品・タイマー・不良入力が本端末で操作可能になります。
3. 必要なら一時停止／休憩の続きから作業します。
4. 常駐カメラは、再開後に **計測中** であれば再び表示されます（一時停止／休憩のまま再開した場合はカメラは出ず、**離席中** が表示されます）。

## 15. 他端末ロックと引き継ぎ

同じ検査生産を別のタブレットで開いている場合、端末ロックにより操作が制限されます。

| 状況 | 表示・ボタン | 操作 |
|------|--------------|------|
| 他端末で生産中 | 「他端末生産中」 | 原則その端末で継続 |
| 自分担当・他端末ロック | **本端末で引き継ぐ** | 確認後、ロックを本端末へ |
| 編集権限あり・強制解除 | **ロック解除** | 確認後、ロックのみ解除（運用ルールに従う） |

引き継ぎ確認ダイアログの趣旨：

> 他端末のロックを解除して、本端末で操作を再開しますか？

**注意**：強制解除は現場リーダー等、権限のあるユーザーのみ使用してください。

## 16. オフラインと自動同期

通信が切れると、画面上部にオレンジ系のバナーが表示されます。

![オフライン表示](./images/InspectionActualAndroid/17_offline.png)

| 表示 | 意味 |
|------|------|
| オフライン作業中… | 変更は端末に保存。接続復帰後に自動アップロード |
| 同期待ち N 件 | 未送信キューの件数 |
| 再読込 | 同期再試行とサーバー再取得 |

**ポイント**

- オフライン中もタイマー・不良入力は継続できます。
- 箱 QR のサーバー登録や **QR読取履歴** の取得にはオンラインが必要です。オフライン時は履歴が表示できない／登録に失敗する場合があります。
- 接続が戻ると自動でアップロードされ、「オフライン変更をサーバーにアップロードしました」等のメッセージが出ます。
- 長時間オフラインのあとは、確定内容がサーバーに反映されたか一覧で確認してください。

## 17. 言語切替とアプリ内ヘルプ

### 17.1 言語

ヘッダー右の **JP / EN / ZH / VI** で表示言語を切り替えます。ボタンラベル・メッセージが切り替わります。

### 17.2 アプリ内ヘルプ（？）

右上の **？** をタップすると、短い操作要約が表示されます。詳細は **本書（Web マニュアル）** を参照してください。

主なポイント：

1. 生産日・検査製品を選び「生産開始」で計測を開始します。  
2. 一時停止・休憩・再開で稼働時間を記録します。一時停止／休憩中はカメラが止まり、下部に **離席中** が表示されます。  
3. 不良は項目別に +/- で入力します（生産中のみ）。  
4. **計測中**は右側の常駐カメラで箱 QR を読み取れます（製品一致時のみ登録）。再開／休憩再開でカメラが再び起動します。  
5. **QR読取履歴** で当該計画の読取一覧と合計を確認できます。  
6. 「生産終了」で QR 読取合計を確認し、箱数/本数を確定して実績を保存します。  
7. 「次の製品（参考）」は「選ぶ」で反映でき、反映後は表示がクリアされます。  
8. オフライン時は端末に保存され、接続復旧後に自動同期されます。

## 18. よくあるメッセージと対処

| メッセージ（要旨） | 対処 |
|--------------------|------|
| 生産中は製品を変更できません… | 先に生産終了する |
| 検査製品を選択すると作業を開始できます | 製品を選ぶか読取する |
| 先に検査製品を選択してください（QR履歴） | 製品を選んでから **QR読取履歴** を開く |
| 5 桁以外／製品が見つからない（読取） | コードとマスタを確認 |
| **QR読取成功　…**（一行表示） | 正常。箱 1 件がサーバーへ登録された |
| **製品不一致** | 生産中製品と QR の製品が違う。正しい箱の QR を当てる |
| **読取間隔**（あと ○ 秒…） | 成功読取の直後。表示どおり待ってから再読取 |
| QR読取履歴の取得に失敗しました | 通信・ログイン権限・バックエンド起動を確認し、再読込 |
| 他の検査員の実績は編集できません | 本人アカウントでログイン |
| この検査員は別製品を生産中です… | 既存の在産を再開するか終了する |
| 他端末で生産中です… | 引き継ぎ／ロック解除／元端末で継続 |
| サーバーと同期できません… | 通信・VPN・**再読込** |
| 不良項目が未登録です | マスタ担当へ連絡 |
| カメラ権限が必要です／起動失敗 | 端末のカメラ権限を許可し、必要ならアプリを再起動 |
| 画面下部に **離席中** | 一時停止中または休憩中。席に戻ったら **再開**／**休憩再開** |
| 常駐カメラが出ない（在産なのに） | 位相が一時停止／休憩でないか確認。計測中に戻すと再表示される |

## 19. 運用上の注意

1. **作業開始前にログインユーザーを確認**（共有端末では特に重要）。
2. 生産中にアプリを強制終了しても、サーバー上は在産のまま残ることがあります。再開時は **作業を再開** を使ってください。
3. 通信断が続くと、検査モニタ上で「通信断」と表示されることがあります。不要な在産はモニタ側の無効化、または正しい終了操作で解消します。
4. Device Owner（キオスク）運用の端末では、アプリの再インストール手順が通常と異なる場合があります。IT 担当の手順に従ってください。
5. 実績の最終確認は、必要に応じて Web の検査実績収集／検査モニタでも行えます。
6. **箱 QR 読取** は補助機能です。最終の確定箱数・本数は、現場実数と **QR読取合計／履歴** を照合したうえで入力してください。
7. QR 合計・履歴は **計画 ID ＋ 生産開始〜終了** で絞られます。別セッションで読んだ QR は含まれません。
8. 常駐カメラは **計測中のみ** 表示されます。一時停止／休憩では意図的に止まり、下部に **離席中** が出ます。終了後にカメラが残る場合はアプリを再表示するか、セッション状態を確認してください。
9. 離席するときは **一時停止** または **休憩** を押し、カメラが止まっていることを確認してください（省電力・誤読取防止）。

## 20. Web 版との主な違い

| 項目 | Android アプリ | Web（ブラウザ） |
|------|----------------|-----------------|
| 操作端末 | 現場タブレット向けタッチ UI | PC／タブレットブラウザ |

| データ | **同一サーバー実績**（相互に見える） | 同左 |
