# 検査実績収集 画面 操作説明

| 項目 | 内容 |
|------|------|
| メニュー | **MES実績収集** → **検査実績収集** |
| 画面名（画面上） | **検査実績収集** |
| 操作説明画面 | 画面上部 **マニュアル** ボタン → 左メニュー **MES関連** → **検査実績収集**（`/manuals/inspection-actual`） |

## 1. 画面の目的とできること

検査実績収集画面では、次の業務を **1 製品 × 1 検査員（ログインユーザー）× 1 生産日** 単位で記録します。

| 機能 | 説明 |
|------|------|
| 生産タイマー | 生産開始・一時停止・休憩・再開の時刻と、稼働時間・一時停止累計・休憩累計を記録 |
| 不良入力 | 工程マスタ（KT09）に登録された不良項目ごとに **+ / −** で数量入力 |
| 実績確定 | 生産終了時に **箱数** または **生産数（本数）** を入力して確定。サーバーへ保存 |
| 当日履歴 | 同一生産日で確定済みの実績を一覧表示・印刷 |
| 事後修正 | 自分が検査員の確定実績について、生産数・不良・時刻等を修正（オンライン時） |
| 製品読取 | 5 桁数字のバーコード / QR から検査製品を自動選択 |
| オフライン | ネット切断中も計測継続。復帰後に未送信データを自動同期 |
| 次製品参考 | 担当者が検査モニタで指定した「次の製品」を参考表示し、ワンタップで製品選択 |

**対象製品の条件（システム側フィルタ）**

- 製品 CD の **末尾が `1`**（検査用製品コード）
- 製品名に **「加工」「アーチ」** を含むものは一覧から除外
- 製品名の昇順でドロップダウン表示

**検査員について（重要）**

- 検査員は **ログイン中のユーザーに固定** されます（プルダウンで他ユーザーを選ぶことはできません）。
- 確定実績の **編集** は、原則として **自分が検査員として記録した行のみ** 可能です。

## 2. 画面へのアクセスと操作説明の開き方

### 2.1 検査実績収集画面を開く

1. Smart-EMAPs に **検査員としてログイン** します。
2. 左サイドメニューから **MES実績収集** を展開します。
3. **検査実績収集** をクリックします。

![メニューから検査実績収集を開く](./images/InspectionActualDataCollection/02_menu_navigation.png)

### 2.2 本操作説明（マニュアル）を開く

検査実績収集の操作説明は、メイン画面共通の **マニュアル** から開きます。

1. 画面上部ヘッダー右側の **マニュアル** ボタン（本アイコン）をクリックします。
2. **新しいタブ** でマニュアルホームが開きます。
3. 左サイドバーで分類 **「MES関連」** を探し、**「検査実績収集」** をクリックします。

操作説明画面では次の操作ができます。

| ボタン | 機能 |
|--------|------|
| **印刷**（左サイドバー下部） | ブラウザの印刷ダイアログで本書を印刷 |

## 4. ツールバー（上部）

ツールバーは画面上部に固定表示され、**生産日**・**検査員**・**検査製品** の 3 項目を扱います。

![ツールバー](./images/InspectionActualDataCollection/03_toolbar.png)

### 4.1 生産日

| 操作 | 説明 |
|------|------|
| 日付ピッカー | 対象の生産日（`YYYY-MM-DD`、日本時間基準）を選択 |
| **◀（前日）** | 選択日を 1 日前へ |
| **今日** | 当日（JST）にジャンプ |
| **▶（翌日）** | 選択日を 1 日後へ |

**注意**

- 生産日を変更すると、その日の検査指示・確定実績・進行中セッションが再読込されます。
- 不正な日付の場合は「生産日が不正です」と表示されます。

### 4.2 検査員（ログインユーザー固定）

| 項目 | 説明 |
|------|------|
| 表示欄 | **ログイン中のユーザー名** が読み取り専用で表示されます |
| 変更 | **不可**。他の検査員名義で記録する場合は、その検査員アカウントでログインし直してください |

**運用上のポイント**

- 共有端末を使う現場では、作業開始前に **正しい検査員でログインしているか** 必ず確認してください。
- 未ログイン状態では実績収集を開始できません（「ログイン後に実績収集を行ってください」）。

### 4.3 検査製品

| 項目 | 説明 |
|------|------|
| プルダウン | `製品CD · 製品名` 形式で選択。タッチ端末では検索（filterable）がオフになる場合があります |
| クリア（×） | 製品の選択を解除 |
| **読取** ボタン（カメラアイコン） | バーコード / QR 読取ダイアログを開く（[§15](#15-バーコード--qr-読取)） |
| 無効化 | **生産中**（生産開始後・終了前）は製品変更不可 |

製品未選択時は、作業エリアに次のメッセージが表示されます。

> 「検査製品を選択すると作業を開始できます」

![製品未選択](./images/InspectionActualDataCollection/05_empty_select_product.png)

### 4.4 生産カードが表示される条件

**検査製品** を選択すると、中央に **生産カード** が表示されます（検査員はログインユーザーに自動紐づけ）。

同一製品で当日すでに実績確定済みでも、再度製品を選べば **新規の生産セッション** として記録できます（履歴は下部テーブルに残ります）。確定済みのみの製品を選んだ場合は、カードの代わりに「本日すでに実績確定済みです。下の一覧をご確認ください」と案内されることがあります。

## 5. 検査生産中パネル

当日・未終了の検査生産が 1 件以上ある場合、ページタイトル横の **リストアイコン（件数バッジ付き）** が表示されます。クリックすると右側から **検査生産中パネル（ドロワー）** が開きます。

![検査生産中パネル](./images/InspectionActualDataCollection/06_in_progress_strip.png)

| 表示要素 | 説明 |
|----------|------|
| 製品名 | 検査対象製品 |
| 検査員名 | 担当検査員。未設定時は「未選択」 |
| 状態ラベル | 計測中 / 一時停止中 / 休憩中 / **他端末生産中** など |
| **作業を再開** | 本端末ロック、またはロック未設定の在産行で操作を再接続（[§16](#16-複数端末セッション復旧)） |
| **本端末で引き継ぐ** | 他端末ロック中だが **ログイン検査員が担当** のとき、ロックを引き継ぐ（[§16.3](#163-検査生産中パネルでの操作)） |
| **ロック解除** | 他端末ロック中で引き継ぎ不可のとき、編集権限者が強制解除（[§16.3](#163-検査生産中パネルでの操作)） |

**操作手順**

1. タイトル横の **リストアイコン** をタップします。
2. パネル内の行をタップ → その製品・検査生産にフォーカス（ツールバーの製品が連動）。
3. 必要に応じて **作業を再開** / **本端末で引き継ぐ** / **ロック解除**（状況に応じて表示）を実行します。
4. パネル外をタップするか、閉じる操作でドロワーを閉じます。

**同期**

- 進行中一覧は約 **5 秒ごと** にサーバーと同期されます（ブラウザタブが非表示のときは間隔が長くなります）。

## 6. 次の製品（参考）指定

担当者が **検査モニタ** 画面で、稼働中または未稼働の検査員に対して「次に検査する製品」を指定した場合、作業エリア上部に **「次の製品（参考）」** バーが表示されます。

| 表示要素 | 説明 |
|----------|------|
| 製品 | `製品CD · 製品名`（参考表示） |
| **選ぶ** ボタン | ワンタップで検査製品プルダウンにその製品を設定（生産中・製品ロック時は無効） |

**注意**

- 担当者指定は **参考情報** です。自動で生産開始はしません。
- **選ぶ** を押さなくても、手動でプルダウンから同じ製品を選べます。
- 約 5 秒ごとにサーバーと同期し、指定の更新・解除が反映されます。

**担当者側（検査モニタ）の操作**

詳細は **[検査モニタ 操作説明](./inspection-monitor_ja.md)** §10 を参照してください。

1. メニュー **MES** → **モニタリング** → **検査モニタ** を開く
2. 対象検査員カードで **次製品指定** または **初回製品を指定** を押す
3. 製品を選択して **保存**（解除は **指定解除**）

## 7. オフライン・未同期の表示

ネットワーク状態に応じて、ツールバー下にオレンジ系の **ステータスバー** が表示されます。

![オフライン表示](./images/InspectionActualDataCollection/17_offline_banner.png)

| 表示文言（例） | 意味 |
|----------------|------|
| オフラインです。計測は継続でき… | ブラウザがオフライン。端末に保存し、復帰後に同期 |
| 未同期 {n} 件。接続中に自動送信… | オンラインだが、サーバー未送信の PATCH がキューに残存 |

**運用**

- 一時停止・休憩・不良入力・タイマー計測はオフラインでも継続可能です。
- **生産終了の確定** と **確定実績の修正保存** は **オンライン必須** です。

詳細は [§17 オフライン動作と自動同期](#17-オフライン動作と自動同期) を参照してください。

## 8. 生産カード（メイン作業エリア）

生産カードは、選択中の製品に対する **タイマー・生産操作ボタン・不良パネル** をまとめたエリアです。

### 8.1 カード上部

| 要素 | 説明 |
|------|------|
| 製品 CD タグ | 例：`XXXXXXXX1` |
| 状態タグ | **未開始** / **計測中** / **一時停止中** / **休憩中** / **終了済** |
| 製品名 | 読みやすい大きめの表示 |
| 不良合計 | 現在セッションの不良数量合計 |
| 検査員 | ログインユーザー名（読み取り専用） |
| 次の製品（参考） | 担当者指定がある場合、カード内にも同様のバーが表示されます |

### 8.2 タイマー表示

![未開始](./images/InspectionActualDataCollection/07_production_card_idle.png)
![計測中](./images/InspectionActualDataCollection/08_production_card_running.png)
![一時停止](./images/InspectionActualDataCollection/09_production_card_paused.png)

| 表示項目 | 説明 |
|----------|------|
| **稼働時間** | 生産開始からの **正味稼働時間**（`HH:MM:SS`）。一時停止・休憩中はカウントが止まります |
| フェーズ | 未開始 / 計測中 / 一時停止中 / 休憩中 |
| **一時停止** | 一時停止の累計時間 |
| **休憩** | 休憩の累計時間 |
| 生産開始 | 開始日時（未設定は `—`） |
| 生産終了 | 終了確定後に表示 |

**一時停止リマインド**

- 一時停止が **30 秒ごと** に続くと、**再開** ボタンが一時的に強調表示されます（見落とし防止）。

### 8.3 生産操作ボタン

画面上のボタンは左から次の順序で並びます。

| ボタン | 色・意味 | 有効条件（概要） |
|--------|----------|------------------|
| **生産開始** | 緑系 | ログイン済み・製品選択済み・未開始・他製品ロックなし・編集権限あり |
| **一時停止** | 琥珀系 | 計測中かつ本端末が操作中 |
| **再開** | 緑系 | 一時停止中かつ本端末が操作中 |
| **生産終了** | 赤系 | 生産開始後かつ **一時停止・休憩中でない** こと、本端末が操作中 |
| **休憩** | 茶系（コーヒーアイコン） | 計測中かつ本端末が操作中 |
| **休憩再開** | 緑系 | 休憩中かつ本端末が操作中 |

ボタンが無効（グレー）のときは、[§21](#21-よくあるメッセージと対処) を確認してください。

**生産終了が押せない場合**

- **一時停止中**：先に **再開** してください（ツールチップ：「一時停止中は生産終了できません」）。
- **休憩中**：先に **休憩再開** してください。

## 9. 標準作業フロー

以下は、1 ロット（1 製品）の検査実績を記録する **推奨手順** です。

![検査実績収集 標準作業フロー](./images/InspectionActualDataCollection/19_workflow.png)

### ステップ 1：開始前の準備

1. **正しい検査員アカウントでログイン** していることを確認します。
2. **生産日** が現場の作業日と一致しているか確認（必要なら「今日」ボタン）。
3. 担当者から **次の製品（参考）** がある場合は、**選ぶ** またはプルダウンで製品を指定。
4. **検査製品** をプルダウンまたは **読取** で指定。

### ステップ 2：生産開始

1. **生産開始** を押します。
2. 「生産を開始しました」と表示され、タイマーが **計測中** になります。
3. サーバーに `関連データベース` 等が記録されます。

### ステップ 3：作業中

- 短い待機・検査中断時は **一時停止** → **再開**。
- 昼休みなど計画的な休憩は **休憩** → **休憩再開**（[§10](#10-一時停止と休憩の違い)）。
- 不良が発生したら、下部の不良パネルで **+** を押してカウント（[§11](#11-不良項目別の入力)）。
- 製品の変更は **できません**。変更する場合は先に **生産終了** して確定してください。

### ステップ 4：生産終了・確定

1. **一時停止・休憩を解除** した状態で **生産終了** を押す → 確認ダイアログが開きます（[§12](#12-生産終了実績確定ダイアログ)）。
2. 表示された開始・終了時刻・稼働時間・不良を確認。
3. **箱数** または **生産数（本数）** を入力（入数マスタがある製品は連動計算）。
4. **実績を確定** を押す。
5. 「実績を保存しました」と表示され、下部 **本日の確定実績** に行が追加されます。

### ステップ 5：次の製品へ

- ツールバーで別の **検査製品** を選び、同じ検査員（ログインユーザー）で繰り返します。
- 別の検査員で記録する場合は、**ログアウトしてその検査員でログイン** してください。

## 10. 一時停止と休憩の違い

検査実績収集では、待機時間を **一時停止** と **休憩** の 2 種類で記録できます。いずれも稼働時間のカウントは止まりますが、集計・運用上の意味が異なります。

| 項目 | 一時停止 | 休憩 |
|------|----------|------|
| ボタン | **一時停止** / **再開** | **休憩** / **休憩再開** |
| 想定用途 | 短い待ち・部品待ち・検査中断 | 昼休み・計画的な休憩 |
| タイマー表示 | 「一時停止」累計が増加 | 「休憩」累計が増加 |
| 状態タグ | 一時停止中 | 休憩中 |
| 生産終了 | **不可**（先に再開が必要） | **不可**（先に休憩再開が必要） |
| 同時操作 | 計測中のみ一時停止可能 | 計測中のみ休憩開始可能 |

**サーバー保存**

- 一時停止累計（秒）と休憩累計（秒）は別フィールドで保存されます。
- 履歴テーブルの「一時停止(分)」列には、一時停止と休憩を合算した値が表示される場合があります（修正ダイアログでは秒単位で調整可能）。

## 11. 不良（項目別）の入力

不良パネルは生産カード下部の **「不良（項目別）」** です。

![不良ステッパー](./images/InspectionActualDataCollection/10_defect_stepper.png)

> 帰属工程ごとに不良項目がグループ化されます。**検査** 工程の項目は **1 行** で表示され、画面幅が狭い場合は横スクロールで確認できます。

| 操作 | 説明 |
|------|------|
| **−** ボタン | 該当項目の不良を 1 減らす（0 未満にはならない） |
| 中央の数字 | 現在の不良数 |
| **+** ボタン | 該当項目の不良を 1 増やす |

**ルール**

- **生産開始後** のみ入力可能（未開始・他端末ロック時は無効）。
- 項目名は **工程別不良項目マスタ**（検査工程 `KT09`）から自動取得。
- マスタ未登録時は「不良項目が未登録です（…KT09 を登録…）」と表示されます。
- 不良の変更はデバウンス後にサーバーへ PATCH されます（オフライン時はキューに積まれます）。

**不良合計**

- カード上部の **不良合計** に、全項目の合計がリアルタイム反映されます。
- 確定時、項目別の内訳は `mes_defect_by_item` として JSON 保存されます。

## 12. 生産終了・実績確定ダイアログ

**生産終了** 押下後に表示されるモーダルです。

![生産終了ダイアログ](./images/InspectionActualDataCollection/11_end_dialog.png)

### 12.1 確認内容

| 項目 | 説明 |
|------|------|
| 製品 CD・製品名 | 対象製品 |
| 検査員 | 確定時の検査員名（ログインユーザー） |
| 生産開始 / 生産終了 | 終了操作時点の時刻 |
| 稼働時間 | 正味稼働時間（開始〜終了 − 一時停止 − 休憩） |
| 不良合計 | 0 より大きい場合のみ表示 |
| 不良（項目別） | チップ形式で内訳表示 |

### 12.2 生産数の入力（箱数・本数）

製品マスタに **入数（本/箱）** が設定されている場合、次の 2 方式で入力できます。

| 入力欄 | 説明 |
|--------|------|
| **箱数** | 箱数を入力すると、**生産数 = 箱数 × 入数** で自動計算されます |
| **生産数（本数）** | 本数を直接入力すると、**箱数 = 本数 ÷ 入数（四捨五入）** で参考表示されます |

**入数未設定の製品**

- 入数が 0 または未設定の場合は、**生産数（本数）のみ** 直接入力します。

**割り切れない本数**

- 本数が入数で割り切れない場合、警告メッセージが表示されます（例：「本数 25 は入数 12 で割り切れません」）。
- 箱数は参考表示（四捨五入）となりますが、**入力した本数のまま確定** することも可能です（確認ダイアログが出る場合があります）。

### 12.3 確定操作

1. 箱数または本数を入力します。
2. **実績を確定**（プライマリボタン）をクリック。  
   - Enter キーでも確定できます。
3. **キャンセル** でダイアログを閉じ、計測を継続できます（終了時刻は確定まで確定しません）。

**制約**

- **オフライン時は確定不可**。「生産終了の確定はオンライン時のみ…」と表示されます。
- 生産数が不正（空・負数・数値以外）の場合は警告されます。

## 16. 複数端末・セッション復旧

検査実績は、サーバー上の **端末ロック** により、同一在産行を **1 端末ずつ** 操作する仕組みになっています。タブレットのブラウザ更新・別端末への移動・通信断からの復帰時に、本書の手順で操作を再接続してください。

![他端末生産中](./images/InspectionActualDataCollection/18_other_terminal_lock.png)

### 16.1 端末ロックの仕組み

| 項目 | 説明 |
|------|------|
| 端末 ID | ブラウザごとに生成され、**ログインユーザー単位** で端末ストレージに保持されます（`mes_client_instance_id`） |
| ロック取得 | **生産開始** や **本端末で引き継ぐ** 成功時に、サーバー行へ端末 ID が記録されます |
| checkpoint | 生産中は **約 5 秒ごと** にタイマー・一時停止・休憩・不良の状態を PATCH（サイレント送信可） |
| 最終アクティビティ | checkpoint 成功時に `mes_client_lock_activity_at` が更新されます（マイグレーション 52 適用後） |
| 自動解放 | 最終アクティビティ（または生産開始時刻）から **4 時間** 超過した在産行は、サーバーがロックを自動解除します |
| リアルタイム同期 | WebSocket `mes_inspection_state` 受信時、および **5 秒（タブ非表示時 15 秒）** ごとに一覧を再取得します |

**ロック状態（本端末から見た判定）**

| 状態 | 意味 | 生産操作 |
|------|------|----------|
| **本端末（mine）** | 自分の端末 ID がロック中 | 一時停止・休憩・不良・生産終了が可能 |
| **他端末（other）** | 別端末（または以前のブラウザ）がロック中 | 原則操作不可。引き継ぎまたはロック解除を検討 |
| **未設定（unclaimed）** | サーバー上は在産だがロック ID なし | **作業を再開** で本端末を再接続 |

### 16.2 画面上の通知

生産カード表示中、サーバー在産状態に応じて次のアラートが出ます。

![セッション復旧](./images/InspectionActualDataCollection/16_session_recovery_alert.png)

#### 16.2.1 セッション復旧アラート（青・情報）

**表示条件（概要）**

- 選択中の製品がサーバー上 **在産中**
- ロックが **他端末ではない**
- 本端末がまだ「操作中」として記録されていない（ページ更新・オフライン復帰など）

**内容**

- タイトル：「未終了の検査生産があります（更新・オフラインで表示が途切れた可能性）」
- **作業を再開** ボタン → [§16.4](#164-作業を再開) と同じ処理

#### 16.2.2 他端末ロックバナー（橙・警告）

**表示条件**

- 選択中の製品がサーバー上 **在産中**
- ロック所有者が **他端末**

**メッセージとボタン**

| 条件 | バナー文言 | ボタン |
|------|------------|--------|
| ログイン検査員が当該行の担当者 | 「別端末（または以前のブラウザ）でロック…引き継ぎ可能」 | **本端末で引き継ぐ** |
| 上記以外（別検査員の在産など） | 「他の端末で操作中です」 | **ロック解除**（`canEdit` 権限がある場合のみ） |

### 16.3 検査生産中パネルでの操作

タイトル横 **リストアイコン** から開くドロワーでも、各行に状態とボタンが表示されます。

| ボタン | 表示条件 | 動作 |
|--------|----------|------|
| **作業を再開** | 本端末ロック、またはロック未設定の在産行 | [§16.4](#164-作業を再開) |
| **本端末で引き継ぐ** | 他端末ロック **かつ** ログイン検査員が担当 | 確認後、ロックを本端末へ移譲 |
| **ロック解除** | 他端末ロック **かつ** 編集権限あり | 確認後、サーバー上のロックのみ解除 |

> 他端末ロック中で **担当検査員でない** 場合、引き継ぎ・再開ボタンは出ず、現場リーダー等が **ロック解除** するか、元端末での終了を待ちます。

### 16.4 作業を再開

**対象**：ロック未設定、またはすでに本端末ロックの在産行。

1. **作業を再開**（青アラート・生産中パネル）を押します。
2. ログインユーザーが検査員として行と一致していることを確認します。
3. 成功すると「本端末の操作を再開しました。一時停止・生産終了が可能です。」と表示されます。
4. タイマーがサーバー時刻に合わせて再計算され、**一時停止・休憩・生産終了・不良入力** が有効になります。

**失敗例**

| メッセージ | 原因 |
|------------|------|
| サーバー上に進行中の検査生産はありません… | すでに終了済み。端末の古い状態をクリア |
| この検査生産は他の端末で生産中です… | 他端末ロックかつ引き継ぎ不可 |

### 16.5 本端末で引き継ぐ（ロック移譲）

**対象**：他端末がロック中だが、**ログイン中の検査員** がその在産行の担当者である場合。

1. 橙バナーまたは生産中パネルの **本端末で引き継ぐ** を押します。
2. 確認ダイアログ：「本端末で操作を引き継ぎますか？他端末（または以前のブラウザ）の操作接続は切断されます。」
3. **本端末で引き継ぐ** で確定 → サーバーへ `mes_claim_client_lock` を送信。
4. 成功後、本端末で通常の生産操作が可能になります。

**運用上の注意**

- 引き継ぎ後、**以前の端末** では一時停止・終了等ができなくなります（409 競合時はサーバー状態を再取得）。
- 共有タブレットで **正しい検査員でログイン** していることが前提です。

### 16.6 ロック解除（強制）

**対象**：他端末ロック中で、引き継ぎできない場合。**編集権限（`canEdit`）** があるユーザーのみボタン表示。

1. **ロック解除** を押します。
2. 確認：「他端末のロックを解除しますか？検査員が本端末から再開できるようになります。」
3. 確定 → サーバーで `mes_force_release` / `mes_release_client_lock` を実行。
4. 「端末ロックを解除しました。」と表示。担当検査員は **作業を再開** または **本端末で引き継ぐ** から再開できます。

> **MES実績クリア** とは別操作です。ロック解除は **在産状態・タイマーは維持** し、操作端末の紐づけのみ外します。

### 16.7 検査員・製品の制約

- 1 検査員（ログインユーザー）は **同時に 1 製品のみ** 在産できます。
- 別製品が在産中のとき、新規 **生産開始** はできません（「この生産日では他の製品が検査生産中です」）。
- **引き継ぎ** は「ログイン検査員 ＝ 行の `mes_inspector_user_id`」のときのみ可能です。

### 16.8 その他

- **画面消灯抑止**：在産中は Wake Lock により画面スリープを抑制します（対応ブラウザのみ）。
- **409 競合**：他端末が先に更新した場合、警告表示後にサーバー状態を再同期します。

## 17. オフライン動作と自動同期

ネットワーク断・サーバー不通時も **端末上の計測は継続** できます。変更内容は端末に保存され、オンライン復帰後にサーバーへ送信されます。

![オフライン表示](./images/InspectionActualDataCollection/17_offline_banner.png)

### 17.1 ステータスバー（ツールバー下）

| 表示 | 条件 | 意味 |
|------|------|------|
| オレンジ点滅＋「オフラインです。計測は継続でき…」 | ブラウザがオフライン | `navigator.onLine` が false |
| 「未同期 {n} 件。接続中に自動送信…」 | オンラインだが送信待ちキューあり | ローカルキューに PATCH が残存 |

いずれも **計測タイマー自体は止まりません**（生産カード上の稼働時間表示は継続）。

### 17.2 オフライン／不通時にできる操作

| 操作 | 端末計測 | サーバー送信 |
|------|----------|--------------|
| 生産開始 | ○ | キューに積む（復帰後に送信） |
| 一時停止 / 再開 | ○ | 同上 |
| 休憩 / 休憩再開 | ○ | 同上 |
| 不良 +/- | ○ | 同上（デバウンス後） |
| タイマー checkpoint（約 5 秒） | ○ | サイレントキュー（通知なしで積む場合あり） |
| **本端末で引き継ぐ** | — | オンライン必須（オフライン時はキューのみ） |
| **生産終了・実績確定** | **×** | **オンライン必須** |
| **確定実績の修正保存** | **×** | **オンライン必須** |
| **ロック解除** | — | オンライン必須 |

**送信できなかった PATCH**

- オフライン、または **5xx / ネットワークエラー** 時は `localStorage` のオフラインキューに保存されます。
- 同一 **生産日 × 計画 ID** の PATCH は **マージ** され、最新のフィールドが優先されます。

### 17.3 オンライン復帰時

1. ブラウザの **online** イベント、または画面再読込時に **未送信キューを自動フラッシュ** します。
2. 成功件数が表示されます：「{n} 件をサーバーに同期しました。」
3. 復帰後は **検査指示一覧を再取得** し、サーバー状態と端末セッションを整合します。
4. 一部失敗時：「一部のデータはまだ同期できません。接続を確認してください。」→ 接続確認後、画面を再読込してください。

**409 競合時**

- 他端末が同じ行を更新した場合、PATCH が拒否されサーバー一覧を再同期します。端末ロック・在産状態を確認してください（[§16](#16-複数端末セッション復旧)）。

### 17.4 ローカル永続化（計測状態）

| 保存先 | 内容 |
|--------|------|
| 端末ストレージ（計測） | 生産日・選択製品・各 plan のタイマー・不良・「本端末操作中」フラグ |
| 端末ストレージ（オフラインキュー） | 未送信 PATCH の一覧 |
| 端末ストレージ（端末 ID） | ログインユーザーごとの `mes_client_instance_id` |

**再訪問時**

- 同一生産日で画面を開き直すと、端末に残っていた計測状態を復元することがあります（「前回の計測状態を復元しました」）。
- サーバー上ですでに終了している在産は、再取得時に **古い端末状態を整理** します。

### 17.5 同期のタイミング（参考）

| トリガー | 動作 |
|----------|------|
| 約 5 秒周期 | 在産中 plan の checkpoint PATCH、サーバー一覧同期 |
| タブ再表示 | 同期間隔を短くし、即時 `syncMesStateFromServer` |
| WebSocket | 同一生産日の検査状態変更通知を受信したとき再同期 |
| 生産日変更・初回読込 | 一覧 GET ＋ オフラインキューフラッシュ |

**確定の原則**

- 公式な在産・確定データは **サーバー同期完了後** の `inspection_management` を正とします。オフライン中は端末表示を参考にし、**必ずオンライン復帰後に未同期件数が 0 であること** を確認してから日報・引き継ぎを行ってください。

## 18. 権限と操作制限

画面の各操作は、ログインユーザーの **MES 操作権限** に応じて制限されます。

| 権限フラグ | 影響する操作 |
|------------|--------------|
| `canCreate` / `canEdit` | 生産開始・一時停止・休憩・不良入力・生産終了 |
| `canEdit` | 確定実績の修正保存 |
| `canDelete` | 確定実績ダイアログの **MES実績クリア** |
| `canExport` | 確定実績一覧の **印刷** |

**検査員と実績の紐づけ**

| ルール | 説明 |
|--------|------|
| 記録時 | 常にログインユーザーが検査員として保存 |
| 修正時 | 自分が検査員の行のみ **編集** リンク表示 |
| クリア時 | 自分が検査員の行のみ **MES実績クリア** 可能（かつ `canDelete`） |

## 20. 指標の計算方法

確定実績テーブルで表示される指標は、画面ロジックに基づき次のとおりです。

### 20.1 不良率

```
不良率(%) = (不良数 ÷ 生産数) × 100
```

- 生産数が 0 以下の場合は `—` 表示。
- 表示は **小数第 1 位**。

### 20.2 能率

```
能率 = 生産数 ÷ (正味稼働時間[時間])
```

- 正味稼働時間 = 壁時計の開始〜終了（秒）− 一時停止累計（秒）− 休憩累計（秒）。
- 結果は **整数** に四捨五入。

### 20.3 稼働時間(分)・一時停止(分)

- 履歴表では **分単位の整数**（秒を 60 で割って四捨五入）。
- 修正ダイアログ内の一時停止は **秒単位** で入力。

## 21. よくあるメッセージと対処

| メッセージ（要約） | 対処 |
|--------------------|------|
| ログイン後に実績収集を行ってください | ログインしてから画面を開き直す |
| 検査員はログイン中のユーザーと一致する必要があります | 共有端末で別ユーザーのセッションが残っている可能性。再ログイン |
| 検査員を選択してください | 未ログイン、または検査員 ID が取得できない状態 |
| 生産中は製品を変更できません | 先に生産終了してから製品変更 |
| 一時停止中は生産終了できません | **再開** してから **生産終了** |
| 休憩中は生産終了できません | **休憩再開** してから **生産終了** |
| この検査員は別製品を生産中です | 該当製品を終了するか、別アカウントでログイン |
| 製品「…」は … が検査生産中のため、開始できません | 他検査員の生産完了を待つ |
| この生産日では他の製品が検査生産中です | 進行中製品を終了してから開始 |
| 生産終了の確定はオンライン時のみ | ネット接続を復旧してから確定 |
| 確定実績の修正はオンライン時のみ | 同上 |
| 自分が検査員の確定実績のみ修正できます | 該当行の検査員本人でログイン |
| 不良項目が未登録です | マスタで KT09 を登録 |
| 製品一覧の取得に失敗しました | 再読込・管理者に連絡 |
| 検査指示の取得に失敗しました | 同上 |
| この検査生産は他の端末で生産中です | 担当検査員なら **本端末で引き継ぐ**、でなければ **ロック解除** または元端末で終了 |
| 本端末で操作を引き継ぎますか？ | 確認ダイアログ。意図どおりなら **本端末で引き継ぐ** |
| 端末ロックを解除しました | 解除成功。担当者が **作業を再開** または **引き継ぎ** |
| サーバーに未送信のため端末に保存しました… | オフライン／不通。計測継続、復帰後に自動同期 |
| 未同期 {n} 件。接続中に自動送信… | オンラインだがキュー残。しばらく待つか再読込 |
| {n} 件をサーバーに同期しました | 同期成功 |
| 一部のデータはまだ同期できません… | 接続・競合を確認し再読込 |
| 前回の計測状態を復元しました | 端末保存から復元。サーバー在産と矛盾ないか確認 |
| 参考製品が検査製品一覧に見つかりません | 担当者指定の製品 CD がマスタにない。手動で近い製品を選択 |

## 22. 運用上の注意
1. **ログインアカウントの確認**  
   検査員はログインユーザーに固定されます。共有タブレットでは **作業のたびに正しいアカウントか確認** してください。

2. **生産日の切り替え**  
   夜勤で日付をまたぐ場合は、切替タイミングを現場ルールで統一してください（「今日」ボタンで誤操作しないよう注意）。

3. **確定の忘れ**  
   生産終了ダイアログで確定しないと、サーバー上は「生産中」のまま残り、他端末・他検査員に影響します。作業終了時は必ず **実績を確定** してください。

4. **一時停止・休憩の使い分け**  
   短い待ちは **一時停止**、計画的な休憩は **休憩** ボタンを使うと、後の分析がしやすくなります。

5. **同一製品の再検査**  
   当日 2 回目以降の検査も可能です。確定実績テーブルに複数行が並びます。

6. **MES実績クリア**  
   誤確定の取り消し用です。使用履歴・権限を運用側で管理してください。

7. **ブラウザ**  
   推奨：Chrome / Edge 最新版。Safari（iOS）でも動作しますが、カメラ・印刷は事前に現場で検証してください。

8. **データの正**  
   公式な集計・報告はサーバー同期完了後の `inspection_management` データを参照してください。
