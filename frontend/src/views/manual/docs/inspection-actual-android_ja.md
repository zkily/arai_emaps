|------|------|

## 1. 画面の目的とできること

検査実績収集では、**1 製品 × 1 検査員（ログインユーザー）× 1 生産日** 単位で次を記録します。

| 機能 | 説明 |
|------|------|
| 生産タイマー | **生産開始** → **一時停止 / 休憩 / 再開** → **生産終了** の時刻と稼働時間を記録 |
| 不良入力 | 不良項目ごとに **＋ / −** で数量入力（生産中のみ） |
| 実績確定 | 終了時に **箱数** または **生産数（本数）** を入力してサーバーへ保存 |
| 当日履歴 | **本日の確定実績** を一覧表示・編集 |
| 製品読取 | カメラでバーコード / QR（5 桁数字）を読み、検査製品を選択 |
| オフライン | 通信切断中も計測継続。復帰後に自動アップロード |
| 次製品参考 | 検査モニタで指定された「次の製品」を表示し、**選ぶ** で反映 |

**検査員について（重要）**

- 検査員は **ログイン中のユーザーに固定** されます（変更不可）。
- 別の検査員名義で記録する場合は、そのアカウントでログインし直してください。
- 確定実績の編集は、原則として **自分が検査員として記録した行のみ** 可能です。

## 2. 事前準備

| 項目 | 内容 |
|------|------|
| 端末 | 現場配付の Android タブレット（SmartEMAP インストール済み） |
| ネットワーク | 社内 Wi‑Fi 推奨。オフラインでも計測可能だが、確定後の同期には接続が必要 |
| アカウント | 検査員本人のユーザーでログイン |
| 製品マスタ | 検査用製品 CD が登録済みであること |
| 不良マスタ | 工程別不良項目（検査工程）が登録済みであること |
| 入数 | 製品の **入数（本/箱）** が設定されていると、終了時に箱数から本数を自動計算できます |

**対象製品の条件（アプリ側フィルタ）**

- 製品名に **「加工」「アーチ」** を含むものは一覧に出ません

## 3. 画面の開き方

1. SmartEMAP アプリを起動し、**検査員アカウントでログイン** します。

![ログイン画面（検査員アカウント）](./images/InspectionActualAndroid/03_login.png)

2. 左上（またはメニューアイコン）からサイドメニューを開きます。
3. **MES管理メニュー** → **実績収集** → **検査実績収集** をタップします。

![メニューから検査実績収集を開く](./images/InspectionActualAndroid/02_menu_navigation.png)

> **注意**  
> 同メニュー内の **「検査実績登録」** は手動一括登録用の別画面です。現場でのリアルタイム計測は必ず **「検査実績収集」** を使ってください。

ダッシュボードにショートカットがある場合は、「検査実績収集」タイルからも開けます。

## 6. ツールバー（生産日・検査員・製品）

### 6.1 生産日

| 操作 | 説明 |
|------|------|
| 日付表示 | 対象の生産日（日本時間基準） |
| **前日** | 1 日前へ |
| **今日** | 当日へジャンプ |
| **翌日** | 1 日後へ |

生産日を変えると、その日の計画・確定実績・進行中セッションが再読込されます。

### 6.2 検査員

| 項目 | 説明 |
|------|------|
| 表示 | ログイン中の氏名（またはユーザー名）が **読み取り専用** で表示 |
| 変更 | **不可** |

### 6.3 検査製品

| 操作 | 説明 |
|------|------|
| プルダウン | `製品CD · 製品名` 形式で選択 |
| **読取** | カメラ読取ダイアログを開く（[§7](#7-バーコード--qr-読取)） |

**生産中は製品変更不可** です。先に生産終了してください。

製品未選択時は次の案内が表示されます。

> 「検査製品を選択すると作業を開始できます」

## 7. バーコード / QR 読取

1. ツールバーの **読取** をタップします。
2. 「バーコード / QR 読取」ダイアログが開きます。
3. 枠内にコードを合わせます。
4. 読み取り成功後、検査製品が自動選択されます。

![バーコード / QR 読取](./images/InspectionActualAndroid/15_barcode_scan.png)

| 項目 | 説明 |
|------|------|
| 読取ルール | **5 桁の数字** のみ有効 |
| 照合 | 製品 CD の末尾が「`{5桁}1`」または「`{5桁}`」の検査製品と照合（`…1` を優先） |
| カメラ | **前面カメラ** / **背面カメラ** を切替可能 |
| 失敗時 | 「読取り再試行」や端末のカメラ権限を確認 |

## 8. 次の製品（参考）

担当者が Web **検査モニタ** で「次の製品」を指定している場合、画面に **次の製品（参考）** が表示されます。

| 操作 | 説明 |
|------|------|
| 表示 | 製品 CD・製品名（参考情報） |
| **選ぶ** | 検査製品にその製品を反映（生産中・ロック時は無効） |

- 自動で生産開始はしません。必ず内容を確認してから **選ぶ**／手動選択してください。
- 詳細な指定方法は **[検査モニタ 操作説明](./inspection-monitor_ja.md)** を参照。

## 9. 生産開始とタイマー操作

製品を選ぶと中央に **生産カード** が表示されます。

### 9.1 生産開始

1. 生産日・製品を確認します。
2. **生産開始** をタップします。
3. タイマーが **計測中** になり、稼働時間がカウントされます。

![計測中の画面](./images/InspectionActualAndroid/08_production_running.png)

| 表示 | 説明 |
|------|------|
| **稼働時間** | 正味の計測時間（一時停止・休憩中は増え方に注意） |
| 位相ラベル | 未開始 / 計測中 / 一時停止中 / 休憩中 / 終了済 |
| 副指標 | 生産開始時刻・一時停止累計・休憩累計 |

### 9.2 操作ボタン一覧

| ボタン | 使えるとき | 動作 |
|--------|------------|------|
| **生産開始** | 未開始 | 計測開始 |
| **一時停止** | 計測中 | 稼働カウントを一時停止 |
| **再開** | 一時停止中 | 計測再開 |
| **休憩** | 計測中 | 休憩カウント開始 |
| **休憩再開** | 休憩中 | 計測へ戻る |
| **生産終了** | 計測中（一時停止・休憩中は不可の場合あり） | 終了ダイアログを開く |

## 10. 一時停止と休憩の違い

| 区分 | ボタン | 使いどころ | タイマーへの影響 |
|------|--------|------------|------------------|
| 一時停止 | **一時停止** → **再開** | 短時間の中断、確認待ちなど | 稼働時間は増えず、一時停止累計が増える |
| 休憩 | **休憩** → **休憩再開** | 休憩時間 | 稼働時間は増えず、休憩累計が増える |

![一時停止中](./images/InspectionActualAndroid/09_production_paused.png)

**注意**

- 一時停止中・休憩中は、状況により **生産終了が無効** になります。先に **再開** / **休憩再開** してから終了してください。
- 一時停止が長い場合、再開ボタンが強調表示されることがあります。

## 11. 不良（項目別）の入力

| 項目 | 説明 |
|------|------|
| 見出し | **不良（項目別）** |
| ヒント | 「生産開始後に +/- で入力」 |
| グループ | **帰属工程** ごとに色分けされたブロック |
| 操作 | 各不良項目の **−** / **＋** |

![計測中の不良入力](./images/InspectionActualAndroid/08_production_running.png)

**ルール**

1. **生産開始後（本端末で操作中）のみ** 増減できます。
2. 数量は 0 未満になりません（− で 0 止まり）。
3. 不良合計は生産カード上部にも表示されます。
4. 不良項目が未登録の場合は「不良項目が未登録です」と表示されます（マスタ担当へ連絡）。

入力内容は随時サーバーへ同期されます（オフライン時は端末に保持）。

## 12. 生産終了・実績確定

1. 計測中に **生産終了** をタップします。
2. 「生産終了・実績確定」ダイアログが開きます。
3. 内容を確認し、数量を入力します。
4. **実績を確定** をタップします。

![生産終了ダイアログ](./images/InspectionActualAndroid/11_end_dialog.png)

### 12.1 ダイアログの項目

| 項目 | 説明 |
|------|------|
| 製品 CD / 製品名 | 確認用（編集不可） |
| 検査員 | ログインユーザー |
| 生産開始 / 生産終了 | 壁時計時刻 |
| 稼働時間 | 正味稼働 |
| 不良合計 | 終了時点の合計（0 より大きいとき表示） |
| **箱数** | 入数設定ありのとき主入力 |
| **生産数** | 入数あり：箱数 × 入数で自動計算／入数なし：本数を直接入力 |

### 12.2 入数がある場合

- 画面に「入数 ○○ 本/箱」と表示されます。
- **箱数** を入れると **生産数** が自動計算されます。
- 本数が入数で割り切れない場合は確認ダイアログが出ます。意図どおりなら **確定する** で続行できます。

### 12.3 入数がない場合

- 「入数未設定（本数を直接入力）」と表示されます。
- **生産数（本数）** を直接入力して確定します。

確定後はタイマーが終了し、下部の **本日の確定実績** に行が追加されます。

## 13. 本日の確定実績と修正

画面下部に、当日・ログイン検査員の確定済み実績が一覧表示されます。

![本日の確定実績](./images/InspectionActualAndroid/12_history.png)

| 列の例 | 説明 |
|--------|------|
| 製品 CD / 製品名 | 対象製品 |
| 箱数 | 入数から換算した箱数（表示） |
| 生産数 | 確定本数 |
| 不良数 / 不良率 | 不良合計と比率 |
| 能率 | 時間あたり生産の指標 |
| 生産開始 / 終了 | 時刻 |
| 稼働時間・一時停止 | 分単位などの表示 |
| 検査員 | 担当者 |
| **編集** | 確定実績の修正ダイアログ |

### 13.1 確定実績の修正

1. 行の **編集** をタップします。
2. 生産数・不良・開始/終了時刻・一時停止秒などを修正します。
3. **保存** でサーバーへ反映します（オンライン推奨）。

> ヒント（アプリ表示）：一時停止累計は秒単位。稼働は開始・終了・停止から自動計算されます。

**他の検査員の実績は編集できません。**

## 14. 検査生産中・セッション復旧

未終了の検査が残っている場合、ヘッダーの一覧アイコンや復旧バナーから操作できます。

| 表示 | 対処 |
|------|------|
| 「未終了の検査生産があります」 | **作業を再開** でタイマーと操作を本端末に接続 |
| 「検査生産中：○○」＋別製品選択中 | **作業を再開** するか、先に生産終了 |

**手順（再開）**

1. **作業を再開** をタップします。
2. 製品・タイマー・不良入力が本端末で操作可能になります。
3. 必要なら一時停止／休憩の続きから作業します。

## 15. 他端末ロックと引き継ぎ

同じ検査生産を別のタブレットで開いている場合、端末ロックにより操作が制限されます。

| 状況 | 表示・ボタン | 操作 |
|------|--------------|------|
| 他端末で生産中 | 「他端末生産中」 | 原則その端末で継続 |
| 自分担当・他端末ロック | **本端末で引き継ぐ** | 確認後、ロックを本端末へ |
| 編集権限あり・強制解除 | **ロック解除** | 確認後、ロックのみ解除（運用ルールに従う） |

引き継ぎ確認ダイアログの趣旨：

> 他端末のロックを解除して、本端末で操作を再開しますか？

**注意**：強制解除は現場リーダー等、権限のあるユーザーのみ使用してください。

## 16. オフラインと自動同期

通信が切れると、画面上部にオレンジ系のバナーが表示されます。

![オフライン表示](./images/InspectionActualAndroid/17_offline.png)

| 表示 | 意味 |
|------|------|
| オフライン作業中… | 変更は端末に保存。接続復帰後に自動アップロード |
| 同期待ち N 件 | 未送信キューの件数 |
| 再読込 | 同期再試行とサーバー再取得 |

**ポイント**

- オフライン中もタイマー・不良入力は継続できます。
- 接続が戻ると自動でアップロードされ、「オフライン変更をサーバーにアップロードしました」等のメッセージが出ます。
- 長時間オフラインのあとは、確定内容がサーバーに反映されたか一覧で確認してください。

## 17. 言語切替とアプリ内ヘルプ

### 17.1 言語

ヘッダー右の **JP / EN / ZH / VI** で表示言語を切り替えます。ボタンラベル・メッセージが切り替わります。

### 17.2 アプリ内ヘルプ（？）

右上の **？** をタップすると、次の要約が表示されます（アプリ内蔵テキスト）。

1. 生産日・検査製品を選び「生産開始」で計測を開始します。  
2. 一時停止・休憩・再開で稼働時間を記録します。  
3. 不良は項目別に +/- で入力します（生産中のみ）。  
4. 「生産終了」で箱数/本数を確定し実績を保存します。  
5. 担当者が指定した「次の製品（参考）」は「選ぶ」で検査製品に反映できます。  
6. オフライン時は端末に保存され、接続復旧後に自動同期されます。  

より詳細な手順は **本書（Web マニュアル）** を参照してください。

## 18. よくあるメッセージと対処

| メッセージ（要旨） | 対処 |
|--------------------|------|
| 生産中は製品を変更できません… | 先に生産終了する |
| 検査製品を選択すると作業を開始できます | 製品を選ぶか読取する |
| 5 桁以外／製品が見つからない（読取） | コードとマスタを確認 |
| 他の検査員の実績は編集できません | 本人アカウントでログイン |
| この検査員は別製品を生産中です… | 既存の在産を再開するか終了する |
| 他端末で生産中です… | 引き継ぎ／ロック解除／元端末で継続 |
| サーバーと同期できません… | 通信・VPN・**再読込** |
| 不良項目が未登録です | マスタ担当へ連絡 |

## 19. 運用上の注意

1. **作業開始前にログインユーザーを確認**（共有端末では特に重要）。
2. 生産中にアプリを強制終了しても、サーバー上は在産のまま残ることがあります。再開時は **作業を再開** を使ってください。
3. 通信断が続くと、検査モニタ上で「通信断」と表示されることがあります。不要な在産はモニタ側の無効化、または正しい終了操作で解消します。
4. Device Owner（キオスク）運用の端末では、アプリの再インストール手順が通常と異なる場合があります。IT 担当の手順に従ってください。
5. 実績の最終確認は、必要に応じて Web の検査実績収集／検査モニタでも行えます。
