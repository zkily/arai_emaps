## 2. 受注（月受注管理）

月次の内示・確定受注を登録・更新し、日別受注データの元となる **月受注** を整備します。後続の生産データ管理（在庫・成型計画列）へ反映される受注情報は、ここで整ったデータが前提です。

### 2.1 画面へのアクセス

1. Smart-EMAPs にログインします。
2. 左サイドメニューから次の順に開きます。  
   **ERP管理メニュー** → **受注管理** → **月受注管理**
3. 画面タイトル **「月受注管理」** が表示されます。

![メニューから月受注管理を開く](./images/FormingPlanningManual/02_menu_order_monthly.png)

### 2.2 画面構成

![月受注管理 画面全体](./images/FormingPlanningManual/07_order_monthly_overview.png)

| エリア | 説明 |
|--------|------|
| ツールバー | 日受注生成、内示更新、製品更新、日受注管理、一括登録 |
| フィルタバー | 年月、納入先、製品キーワード検索 |
| サマリーカード | 内示本数、確定本数、内示差異、社内成型・外注成型 等 |
| 一覧テーブル | 納入先・製品・内示本数・確定本数・操作列 |

### 2.3 初回セットアップ：月受注の登録

#### 2.3.1 一括登録（推奨）

新規月の受注をまとめて登録する場合：

1. ツールバー **「一括登録」** をクリックします。
2. ダイアログで **年・月・納入先** を選択し **「読込」** をクリックします。
3. 納入先に紐づく製品一覧が表示されます。状態が **「新規」** の行について **内示本数** を入力します。
4. 入力後 **「登録」** を実行します。

![一括登録ダイアログ](./images/FormingPlanningManual/09_order_monthly_batch_register.png)

#### 2.3.2 個別編集

一覧から 1 件ずつ修正する場合：

1. 対象行の **「編集」** をクリックします。
2. **基本情報**（納入先・製品・タイプ）と **数量情報**（内示・日合計・差異）を確認・修正します。
3. **「保存」** をクリックします。

行末 **「削除」** で月受注レコードを削除できます（誤登録の修正時）。

### 2.4 日受注リストの生成

月受注から日別内示・確定データを展開します。

1. フィルタで **対象年月** と **納入先** を指定します（未指定の場合は確認ダイアログで表示される条件を確認）。
2. ツールバー **「日受注生成」** をクリックします。
3. 確認ダイアログで **対象年月・製品種別（量産品）・納入先** を確認します。
4. **「生成する」** をクリックし、進捗バーの完了を待ちます。

![日受注生成 確認](./images/FormingPlanningManual/08_order_monthly_daily_generate.png)

> **注意**：日受注生成は、同一条件で再実行すると日別データが再構築されます。運用ルールに従い、必要なタイミングでのみ実行してください。

### 2.5 内示・製品情報の更新

| ボタン | 用途 | 操作概要 |
|--------|------|----------|
| **内示更新** | 内示本数の一括再計算 | 確認ダイアログ → **実行** |
| **製品更新** | 製品マスタ変更の反映 | 開始日を指定 → **製品情報を最新データに更新** にチェック → **更新実行** |
| **日受注管理** | 日別受注の詳細編集 | 別画面（日受注管理）へ遷移 |

### 2.6 サマリーと成型関連の確認

画面上部のサマリーカードで、当該月の **社内成型**・**外注成型** の件数（本数）を確認できます。計画作成前に、対象製品の受注が登録されているか、このサマリーと一覧で照合してください。

### 2.7 受注工程チェックリスト

| # | 確認内容 | OK |
|---|----------|-----|
| 1 | 対象年月・納入先の月受注が登録済みか | ☐ |
| 2 | 内示本数・確定本数に大きな未入力がないか | ☐ |
| 3 | 日受注生成を実行済みか（日別データが必要な場合） | ☐ |
| 4 | 社内成型対象製品が一覧に含まれているか | ☐ |

## 3. 在庫（生産データ管理）

受注・実績・在庫を横断的に管理する画面です。計画作成の前提として、**成型在庫**・**成型前在庫**・**成型計画** 列などが正しく更新されている必要があります。

### 3.1 画面へのアクセス

1. **ERP管理メニュー** → **生産管理** → **生産計画** → **生産データ管理**
2. 画面タイトル **「生産データ管理」**、サブタイトル **「受注・実績・在庫を一元管理」** が表示されます。

![メニューから生産データ管理を開く](./images/FormingPlanningManual/03_menu_data_management.png)

### 3.2 画面構成

![生産データ管理 画面全体](./images/FormingPlanningManual/10_data_management_overview.png)

| エリア | 説明 |
|--------|------|
| ヘッダー | 件数表示、各種更新機能、再取得、印刷、列設定 等 |
| フィルタ | 年月・製品・納入先 等（画面上部） |
| メインテーブル | 日付行 × 工程別列（受注・実績・在庫・計画 等） |
| 下部タブ | 受注／実績／**在庫**／推移／計画 等 — 表示列グループの切替 |

### 3.3 初回：データ生成と受注反映

初めて当該期間の生産データを扱う場合：

1. 右上 **「各種更新機能」** ▼ をクリックします。
2. 次の順序で実行することを推奨します。

| 順序 | メニュー項目 | 目的 |
|------|--------------|------|
| 1 | **データ生成** | 対象期間の行を生成 |
| 2 | **受注データ更新** | 月受注・日受注から受注列を反映 |
| 3 | **初期在庫一括登録** | 期首在庫がある場合のみ |
| 4 | **在庫・推移更新** | 在庫・推移・安全在庫を再計算 |

![各種更新機能メニュー](./images/FormingPlanningManual/12_data_management_update_menu.png)

#### 3.3.1 全部一括更新

上記を個別に実行する代わりに、**「全部一括更新」** 1 回で以下を順に実行できます。

- 受注データ更新
- 繰越データ更新
- 実績・不良・廃棄・保留 更新
- 生産日・設備 更新
- 計画データ更新
- 在庫・推移・安全在庫更新

確認ダイアログの内容を読み、**「一括更新開始」** をクリックします。進捗ダイアログで完了を待ちます。

![全部一括更新（参考）](./images/FormingInstructionManual/14_all_batch_update.png)

### 3.4 在庫 TAB の操作

計画作成前に **成型在庫** の状態を確認します。

1. 画面下部タブ列から **「在庫」**（📦）をクリックします。
2. テーブルに **成型在庫** 等の列が表示されます。
3. 日付行と製品行の交差セルで、在庫数量・推移を確認します。

![在庫 TAB](./images/FormingPlanningManual/11_data_management_inventory_tab.png)

#### 3.4.1 在庫の手動修正（必要な場合）

実物在庫とシステム在庫に差がある場合：

1. 修正したい **日付行** と **工程列**（例：成型在庫）のセルを **ダブルクリック** します。
2. **「在庫取引ログ入力」** ダイアログが開きます。
3. 種別・数量を入力し **登録** します。
4. 修正後、**「在庫・推移更新」** を実行して関連列を再計算します。

![在庫セル ダブルクリック](./images/FormingInstructionManual/16_inventory_tab_dblclick.png)

### 3.5 成型関連列の見方

| 列名 | 意味 |
|------|------|
| 成型前在庫 | 成型直前工程の在庫（投入可能数の参考） |
| 成型在庫 | 成型工程完成品の在庫 |
| 成型実績 | 成型完成実績数 |
| 成型計画 | 計画数量（APS／ERP 連携） |
| 成型実計 | 計画と実績の整合列 |
| 成型推移 | 在庫推移指標 |

### 3.6 在庫工程チェックリスト

| # | 確認内容 | OK |
|---|----------|-----|
| 1 | 受注データ更新後、受注列に数値が入っているか | ☐ |
| 2 | 成型在庫・成型前在庫が妥当か（負数・異常な跳びがないか） | ☐ |
| 3 | 在庫 TAB で対象製品の直近数日を確認したか | ☐ |
| 4 | 修正後は在庫・推移更新を実行したか | ☐ |

### 3.7 成型計画作成（生産データ管理）

受注・在庫が整ったあと、**生産データ管理** 画面上の **「生産計画印刷」** から **成型計画作成** ダイアログを開きます。ダイアログ内では、**計画クリア**・**在庫・推移更新** を **「成型計画作成」（試算）の前** に実行し、そのうえで実計推移に基づく **計画数量の試算** を行います。試算結果は、第 4 章の **APS 成型計画作成** でガント・日別計画に落とし込む際の参考になります。

> **補足**：本ダイアログの **「成型計画作成」** ボタンは、画面上の計算結果を表示・印刷するためのものです。`molding_plan` 列への自動書き込みは行いません。計画列を ERP 表に反映するには **各種更新機能** の **計画データ更新**（または **全部一括更新**）を、日程の確定は APS 側の再計算で行います。

#### 3.7.1 ダイアログを開く

1. 第 3.1 節の手順で **生産データ管理** を開きます。
2. 画面右上 **「生産計画印刷」** ▼ をクリックします。
3. **「成型計画作成」** を選択します。
4. タイトル **「成型計画作成」**（サブ：条件入力・計画クリア・計算結果）のダイアログが表示されます。

![生産計画印刷メニュー（成型計画作成）](./images/FormingPlanningManual/16_data_management_molding_plan_menu.png)

#### 3.7.2 事前準備（画面外）

ダイアログを開く前に、第 3.3 節の **受注データ更新** および **在庫・推移更新**（または **全部一括更新**）まで完了していることを確認してください。設備能率マスタに製品×成型機の能率（本/H）が登録されていることも前提です。

#### 3.7.3 計画クリア・在庫・推移更新（試算の前に実行）

![成型計画作成ダイアログ（条件）](./images/FormingPlanningManual/17_data_management_molding_plan_dialog.png)

ダイアログを開いたら、**「成型計画作成」**（試算ボタン）を押す **前に**、下表の順で操作します。実計推移・対応日の参照は、ここで更新された生産サマリーに基づきます。

| 順序 | 操作 | 内容 |
|------|------|------|
| 1 | **計画クリア実行**（必要時） | **計画クリア開始日** 以降の **成型計画** および **実績計画** を 0 にクリア（確認ダイアログあり）。旧計画を残したまま試算しない場合に実施 |
| 2 | **在庫・推移更新** | メニュー「在庫・推移更新」と同様の再計算（開始日は初期取引ログ基準。**計画クリア開始日とは連動しない**）。**試算の直前に必ず実行**し、成型実計推移を最新化する |
| 3 | マスタ確認（必要時） | **「成型機器設定」**（製品ごとの成型機）、**「製品工程BOM」**（安全在庫日数・成型 LT）。変更は自動保存 |

> **注意**：計画クリアだけ行い **在庫・推移更新** を省略したまま試算すると、実計推移が古いまま計算されることがあります。必ず順序 2 の後に試算してください。

#### 3.7.4 条件入力と試算

| 項目 | 説明 |
|------|------|
| **生産計画月** | 対象月。変更すると **基準日**（翌月 1 日）・**稼働日**（当該月の平日数）・計画クリア開始日の初期値が連動します |
| **基準日** | 対応日計算の起点（通常は生産計画月の翌月 1 日） |
| **稼働日** | 月内平日数（計算結果の **日当り** 用） |
| **加工減耗係数** | 必要数算出に用いる係数（初期値 1.043） |

1. 3.7.3 の **在庫・推移更新** 完了後、上記条件を確認・修正します。
2. **「成型計画作成」**（青色）をクリックして試算します。

**計算の考え方（概要）**

- 各製品の **対応日** ＝ 基準日 ＋（成型 LT ＋ 安全在庫日数）を **営業日** で加算（土日除く）。
- 対応日の `生産管理データベース` 行で **成型実計推移が負** のものだけ対象。
- **必要数** ＝ |実計推移| × 加工減耗係数（四捨五入）→ ロットサイズで切り上げ → **計画数**。
- **能率**・**工時** は設備能率マスタから解決（未登録時は工時「—」）。

対象が無い場合は「対象データがありません」と表示されます（対応日の行が無い、または実計推移が 0 以上）。

#### 3.7.5 計算結果の確認と印刷

計算後、ダイアログ下部に **計算結果** テーブルが表示されます。

![成型計画作成 計算結果](./images/FormingPlanningManual/18_data_management_molding_plan_result.png)

**印刷帳票（サンプル）**

試算で 1 件以上ヒットしたあと、計算結果パネル右上の **「印刷」** をクリックすると、ブラウザの **印刷プレビュー**（別ウィンドウ）が開きます。A4 縦・成型機ごとに改ページしやすいレイアウトで、会議・ライン調整用の帳票として保存・印刷できます。

1. 計算結果が表示されていることを確認します（0 件のときは「印刷するデータがありません」）。
2. **「印刷」** をクリックします。ポップアップがブロックされた場合は、ブラウザで許可してください。
3. プレビュー画面で **成型計画作成 — 計算結果** の内容を確認し、OS の印刷ダイアログから PDF 保存または印刷します。

| 帳票の部位 | 内容 |
|----------|------|
| タイトル | **成型計画作成 — 計算結果** |
| 条件行（meta） | 生産計画月・基準日・稼働日・加工減耗係数（試算時に入力した値） |
| サマリー行（sub） | **計画数合計**（全行の計画数合算）・出力日時・件数 |
| 成型機ブロック見出し | **成型機**名／当該機の **合計計画数**・**合計工時**・**平均日稼働**（合計工時 ÷ 稼働日、時間/日） |
| 明細表 | 対応日・製品名・必要数・ロットサイズ・ロット数・**計画数**（太字）・能率（本/H）・工時 |

画面上の表にある **実計推移**・**製品CD**・**日当り** は帳票には出ません。並び順は **成型機昇順 → 対応日 → 製品CD** です。

![成型計画作成 計算結果の印刷プレビュー（サンプル）](./images/FormingPlanningManual/19_data_management_molding_plan_print_sample.png)

**成型計画推奨生産日（推奨生産日印刷）**

生産データ管理の **「推奨生産日印刷」** から出力する **成型計画推奨生産日** は、画面上の **成型実計推移（molding_actual_plan_trend）** がマイナスの製品について、いつまでに成型すべきかの **推奨日一覧** を成型機別に印刷する機能です。第 3.7.3 節の試算・第 3.7.5 節の試算帳票と同じ **実計推移** を参照するため、計画立案の **日程確認用** リストとして使います。

| 項目 | 成型計画推奨生産日 | 成型推奨生産日（参考） |
|------|-------------------|------------------------|
| メニュー | 推奨生産日印刷 → **成型計画推奨生産日** | 推奨生産日印刷 → 成型推奨生産日 |
| 判定に使う推移 | **成型実計推移** | 成型推移 |
| 帳票タイトル | **成型計画推奨生産日リスト** | 成型推奨生産日リスト |
| 主な列 | 製品名・推奨生産日 | 製品名・推奨生産日・成型前在庫・直前工程 |

**操作手順**

1. **生産データ管理** を開き、第 3.3 節どおり **受注データ更新**・**在庫・推移更新**（または **全部一括更新**）を済ませます。
2. 画面右上 **「推奨生産日印刷」** ▼ をクリックします。
3. **「成型計画推奨生産日」** を選択します。
4. **推奨生産日** 進捗ダイアログでデータ取得が完了するまで待ちます。
5. ブラウザの **印刷プレビュー** が開きます。内容を確認し、PDF 保存または印刷します（ポップアップブロック時はブラウザで許可）。

![推奨生産日印刷メニュー（成型計画推奨生産日）](./images/FormingPlanningManual/20_data_management_rec_molding_plan_menu.png)

**抽出ルール（概要）**

- **対象期間**：画面のデータ取得範囲（当月月初～数か月先、JST）の日別サマリー行。
- **対象製品**：期間内のいずれかの日で **成型実計推移 &lt; 0** の製品（製品ごとに 1 行）。
- **採用行**：**推奨成型生産日（molding_production_date）** が最も早い行を優先（同日なら日付列 `date` が早い方、さらに同順なら実計推移がより小さい方）。
- **並び順**：**成型機** 昇順 → **推奨生産日** 昇順。
- **推奨生産日** 列：採用行の **推奨成型生産日（molding_production_date）**。当日（JST）以前の日付は **赤太字** で強調（着手期限超過・当日の目安）。

**印刷帳票（サンプル）**

| 帳票の部位 | 内容 |
|----------|------|
| タイトル | **成型計画推奨生産日リスト** |
| サブタイトル | 対象期間（月初～取得終了日）／出力日時 |
| 成型機ブロック | 見出し **成型機:** 設備名。2 段組で製品を列挙 |
| 列（各製品） | **製品名**／**推奨生産日**（成型前在庫・直前工程は **出しません**） |

該当製品が無い場合は「該当データがありません」と表示されます。

![成型計画推奨生産日リストの印刷プレビュー（サンプル）](./images/FormingPlanningManual/21_data_management_rec_molding_plan_print_sample.png)

**画面上の計算結果テーブル（列の意味）**

| 列 | 意味 |
|----|------|
| 対応日 | 上記ルールで求めた参照日 |
| 成型機 | 生産サマリー上の成型機名 |
| 製品名／製品CD | 対象製品 |

試算後、ERP 表の **成型計画** 列へ APS 計画を反映する場合は、ダイアログを閉じ、**各種更新機能** → **計画データ更新**（または **全部一括更新**）を実行してください（`schedule_details` から `plan` 列へ集計）。日程の確定は第 4 章 APS の **ライン順で再計算** で行います。

## 4. 計画（成型計画作成）

**APS** 上で成型ラインの生産計画を作成・編集し、ガント表示で日程を確認します。ERP **生産データ管理** で試算した計画数（第 3.7 節）は、ここで計画行として登録し **ライン順で再計算** して日別に展開します。

### 4.1 画面へのアクセス

1. **APS管理メニュー** → **生産計画作成** → **成型計画作成**
2. 画面タイトル **「成型計画作成」** が表示されます。

![メニューから成型計画作成を開く](./images/FormingPlanningManual/04_menu_forming_planning.png)

### 4.2 画面構成

![成型計画作成 画面全体](./images/FormingPlanningManual/13_forming_planning_overview.png)

| エリア | 説明 |
|--------|------|
| セットアップバー | 基準開始月、工程、ライン、計画を取得 |
| 計画追加 | 製品・ロット数／本数・追加先（新規行／既存合算） |
| 計画一覧 | 順位・製品名・合計(本)・開始日・終了日 等 |
| ガント | 日別計画の視覚表示 |
| アンカー日設定 | 再計算開始日の設備別保存 |

### 4.3 計画取得の手順

1. **基準開始月** を設定します（先頭計画の着手月）。
2. **工程** を選択します（成型工程 CD）。
3. **ライン**（成型設備）を選択します。  
   ※ 工程に紐づく設備が設備マスタ・設備能率に登録されている必要があります。
4. **「計画を取得」** をクリックし、既存計画を読み込みます。

必要に応じて **「アンカー日設定」** で設備ごとの再計算開始日を保存できます（保存済みは最優先で使用されます）。

### 4.4 計画の追加

![計画追加フォーム](./images/FormingPlanningManual/14_forming_planning_add_plan.png)

1. **製品名** を選択します（設備能率に登録された製品のみ表示）。
2. 製品選択後、**能率・段取・ロットサイズ・最大日産** がチップ表示されます。
3. **入力方式** を選びます。
   - **ロット数**：ロット単位で追加
   - **本数**：直接本数を指定
4. **追加先** を選びます。
   - **新規行**：計画一覧に新しい行を追加
   - **既存に合算**：同一製品の既存行に数量を加算
5. 数量を入力し **「追加」** をクリックします。

### 4.5 計画一覧の編集

| 操作 | 方法 |
|------|------|
| 順位変更 | 行をドラッグ＆ドロップ（「未完のみ」表示時） |
| 合計(本) 修正 | セルをダブルクリック → 数値入力 → 保存 |
| 開始日指定 | 列「開始日指定」をダブルクリックで日付設定 |
| 削除 | 行末の削除操作 → 確認 |

### 4.6 設備稼働設定

**ライン順で再計算**（第 4.8 節）では、各日の可排産時間を **設備稼働設定**に基づいて計画を展開します。計画数量を入れたあと・再計算の **前** に、対象ライン・対象期間の稼働帯を整えておいてください。

#### 4.6.1 画面へのアクセス

| 経路 | 手順 |
|------|------|
| **メニュー** | **APS管理メニュー** → **設備稼働管理** → **設備稼働設定** |
| **成型計画作成から** | ライン・計画取得後、ガント下 **「設備稼働時間」** 行の **「稼働設定」**、またはセルを **ダブルクリック**（ガント表示期間と同じ期間でダイアログが開く） |

![メニューから設備稼働設定を開く](./images/FormingPlanningManual/22_aps_menu_line_capacity.png)

#### 4.6.2 画面構成と操作

![設備稼働設定 画面全体](./images/FormingPlanningManual/23_aps_line_capacity_overview.png)

| エリア | 説明 |
|--------|------|
| ツールバー | **工程**（切断 KT01／面取 KT02／**成型 KT04**／溶接 KT07）、**設備**、**期間**（今月／次月ショートカット）、**土曜・日曜表示** |
| 一括時間帯 | 表示中の全日にシフトプリセットを一括適用（反映には **一括保存**） |
| 日別カード | 日付・曜日・**実稼働時間（h）**、日ごとのプリセットボタン、時間帯行の追加・削除 |
| 時間帯行 | **開始〜終了**、**休憩**（オンにした行は稼働合計・排産から除外） |
| **一括保存** | 編集内容を DB に反映（成型計画作成のダイアログから開いた場合は保存後にガント側も再取得） |

**成型工程（KT04）の主なプリセット（日カード／一括時間帯）**

| ボタン | 目安 |
|--------|------|
| **4H** | 08:00–12:00 ＋ 10:00–10:10 休憩 |
| **8H** | 昼間 2 直（12:00–13:00 休）＋各直 10 分休憩 |
| **16H** | 昼間＋夜勤帯（21:00–06:00 付近） |
| **20H** / **22H** / **24H** | 長時間・連続稼働パターン（24H は日跨ぎを 2 行に分割して保存） |
| **全日削除** | 表示中日の時間帯をすべてクリア（保存で反映） |

**設定の流れ（成型ライン）**

1. **工程** で **成型（KT04）** を選び、対象 **設備**（成型機）を選択します。
2. **期間** を、再計算したいガント期間（例：基準開始月を含む今月〜翌月）に合わせます。
3. **一括時間帯** または各日のプリセットで時間帯を入れ、必要なら **休憩** を調整します。
4. 右上 **「一括保存」** をクリックし、保存完了を確認します。

> **注意**：「休憩」にチェックした時間帯は、カード上の **○.○h** および再計算の可用時間から差し引かれます。未保存のまま再計算すると、旧い稼働帯で展開されます。

### 4.7 設備稼働時間表

複数設備の **日別稼働時間** を一覧で確認・印刷する画面です。設定漏れや極端に短い／長い日がないか、再計算前の点検に使います。

#### 4.7.1 画面へのアクセス

1. **APS管理メニュー** → **設備稼働管理** → **設備稼働時間表**
2. 画面タイトル **「設備稼働時間表」** が表示されます。

![メニューから設備稼働時間表を開く](./images/FormingPlanningManual/24_aps_menu_capacity_matrix.png)

#### 4.7.2 画面構成と操作

![設備稼働時間表 画面全体](./images/FormingPlanningManual/25_aps_capacity_matrix_overview.png)

| エリア | 説明 |
|--------|------|
| フィルタ | **工程**（初期は成型 KT04）、**設備**（複数選択可）、**期間**、**再取得**、**印刷** |
| マトリクス表 | 行＝設備、列＝日付（土日は列スタイルで区別）。セル＝その日の **稼働時間（時間）** |

5. 会議・掲示用に **「印刷」** で帳票プレビュー（A4）を出せます。

**印刷帳票（サンプル）**

| 帳票の部位 | 内容 |

| 表本体 | 設備名 × 日付列の稼働時間（画面と同じ丸め表示：小数が 0.1 を超えると表示は切り上げ） |

![設備稼働時間表の印刷プレビュー（サンプル）](./images/FormingPlanningManual/26_aps_capacity_matrix_print_sample.png)

### 4.8 ライン順で再計算

計画追加・順位変更・数量修正および **設備稼働設定の保存** のあと、必ず再計算します。

1. 計画一覧右上 **「ライン順で再計算」**（オレンジ）をクリックします。
2. 確認ダイアログの内容を確認します。  
   - 過去日の日別計画は固定  
   - 当日は実績がある場合のみ固定  
   - 開始日指定がある製品は指定日より前に開始しない
3. 実行を承認し、完了メッセージを待ちます。
4. ガントと計画一覧で開始日・終了日・日別数量を確認します。

![ライン順で再計算](./images/FormingPlanningManual/15_forming_planning_replan.png)

## 7. 日次サイクルの繰り返し

初回の **受注 → 在庫 → 計画 → 指示 → 実績** が完了したら、以降は次の **4 ステップ** を毎営業日繰り返します。

```
在庫（確認・更新）→ 計画見直し（再計算）→ 指示（翌日分）→ 実績（収集・照合）
        ↑__________________________________________________________|
                              繰り返し
```

### 7.1 在庫（確認・更新）

1. **生産データ管理** を開き、前日実績反映後の **成型在庫**・**成型実績** を確認します。
2. 生産実績管理で照合・修正が済んでいれば、**「全部一括更新」** または **「在庫・推移更新」** を実行します。
3. 微差がある場合は **在庫 TAB** から取引ログで修正します。

![日次サイクル：在庫と計画](./images/FormingPlanningManual/20_cycle_inventory_replan.png)

### 7.2 計画見直し（再計算）

1. **成型計画作成** または **成型計画一覧**（`/aps/planning-list`）を開きます。
2. 生産残・合計(本) を確認し、必要なら数量を修正します。
3. **「ライン順で再計算」**（計画作成）または **「成型ライン順で再計算」**（計画一覧）を実行します。
4. ガントで残計画の日程を確認します。

詳細手順は [成型指示・実績収集マニュアル 第 9 章](./forming-instruction_ja.md#9-aps-再計算と生産残の修正) を参照してください。

### 7.3 指示（翌日分）

1. **成型指示** で **データ更新** を実行します。
2. 生産日を **翌日** に設定し、**指示書発行** します。
3. 現場へ指示書を配布します。

### 7.4 実績（収集・照合）

1. 現場で HT／MES により実績を登録します。
2. 締め時に **生産実績管理** で照合します。
3. **全部一括更新** で DB へ反映します。
4. 翌朝、**7.1 在庫** からサイクルを再開します。

### 7.5 日次サイクル早見表

| 時期 | 在庫 | 計画 | 指示 | 実績 |
|------|------|------|------|------|
| 前日締め後 | 一括更新 | — | — | 照合完了 |
| 当日朝 | 在庫確認 | 再計算 | — | — |
| 当日昼〜夕 | — | 残数修正 | 翌日指示発行 | — |
| 当日生産中 | — | — | 現場配布 | MES／HT 入力 |
| 当日締め | 更新準備 | — | — | 照合 |

## 8. よくあるメッセージと対処

| メッセージ・現象 | 原因の例 | 対処 |
|------------------|----------|------|
| 対象データがありません（成型計画作成） | 対応日の行なし／実計推移≧0 | 在庫・推移更新、安全在庫・成型 LT、実計推移を確認 |
| 計画データがありません | ライン未選択／未登録 | 工程・ラインを選び計画を追加 |
| この工程に該当する設備がありません | 設備マスタの種別不一致 | 設備マスタ・APS 対象フラグを確認 |
| 製品は工程に属していません | BOM ルートに工程なし | 製品工程 BOM を確認 |
| 順次再計算に失敗 | 稼働カレンダー・能率未設定 | 設備能率・稼働時間を確認 |
| 指示書に計画が出ない | 指示画面が未更新 | **データ更新** を実行 |
| 実績が在庫に反映されない | 一括更新未実行 | 照合後 **全部一括更新** |
| 内示差異が大きい | 日受注未生成・未更新 | 月受注・日受注生成を確認 |
