## 2. 成型指示書の発行（指示書発行）

### 2.1 画面へのアクセス

1. Smart-EMAPs にログインします。
2. 左サイドメニューから次の順に開きます。  
   **ERP管理メニュー** → **生産管理** → **生産指示** → **成型指示**
3. 画面タイトル **「成型指示書発行管理」** が表示されます。

| 操作順 | 

![メニューから成型指示を開く](./images/FormingInstructionManual/02_menu_forming_instruction.png)

### 2.2 画面構成

![成型指示書発行管理 画面全体](./images/FormingInstructionManual/03_forming_instruction_overview.png)

| エリア | 説明 |
|--------|------|
| ヘッダー | 計画生産数・稼働設備数のサマリ、「データ更新」ボタン |
| 検索バー | 生産日、設備選択、キーワード検索、各種印刷ボタン |
| 計画テーブル | 生産日・設備名・製品・計画生産数・能率・段取時間・備考 等 |
| チャート | 日別計画・実績生産数 |

### 2.3 「指示書発行」の操作手順

> **重要**：ツールチップに「基本的には翌日の日付で設定してから発行してください」と表示されています。**原則として翌日の生産日** を選んでから発行してください。

1. **生産日を設定する**
   - 検索バーの **「生産日」** で対象日を選択します。
   - ショートカット **「翌日」** ボタンを押すと、翌日の日付が設定されます（推奨）。

   ![生産日を翌日に設定](./images/FormingInstructionManual/04_forming_date_nextday.png)

2. **必要に応じて絞り込む**（任意）
   - **設備選択**：特定ラインのみ印刷する場合
   - **製品名・設備名検索**：キーワードで計画一覧を絞り込み

3. **「指示書発行」ボタンをクリックする**
   - ボタン位置：検索バー右側、**緑色（success）** の **「指示書発行」** ボタン
   - アイコン：ダウンロード系アイコン付き

   ![指示書発行ボタン](./images/FormingInstructionManual/05_instruction_issue_button.png)

4. **印刷処理の流れ**
   - システムが有効な **成型設備ごとに 1 ページ** の指示書 HTML を生成します。
   - 画面上にプレビューが表示され、**QR コード（製品 CD）の読込** が完了するまで「QRコード読み込み中...」と表示されます。
   - 読込完了後、**ブラウザの印刷ダイアログが自動的に開きます**<span style="color: #dc2626; font-weight: 700;">（A5 横向き）</span>。
   - プレビュー右上の **×** ボタンでキャンセルできます（印刷を中止したい場合）。

   ![指示書印刷プレビュー](./images/FormingInstructionManual/06_instruction_print_preview.png)

5. **プリンタで印刷する**
   - 各設備分の **「成型生産指示書」** を印刷します。
   - 計画データが無い設備は、表内に **「生産計画停止」** 等が表示される場合があります。
   - 印刷後、指示書を成型現場の各ラインへ配布します。

   ![印刷された成型生産指示書](./images/FormingInstructionManual/07_instruction_printed_sample.png)

### 2.4 指示書に含まれる主な情報

- 設備名・設備 CD
- 生産日
- 製品 CD（QR コード付き）
- 生産順位・計画生産数
- 能率・段取時間
- 日別計画一覧（智能日付範囲：前日・当日・翌稼働日 等を自動算出）

### 2.5 補足操作

| ボタン | 用途 |
|--------|------|
| **データ更新** | 画面上部。計画データを再取得 |
| **前日／今日／翌日** | 生産日のクイック切替 |
| **段取表プレビュー** | 第 10 章参照。編集してから印刷 |
| **段取表印刷** | 第 10 章参照。編集せずそのまま印刷 |

## 3. 現場での指示書交換

この工程は **システム画面上の操作ではなく、成型現場での物理的な手順** です。

### 3.1 目的

- 事務側で新規発行した **翌日分の成型工程指示書** を現場へ渡す。
- 現場で **記入・実績記録が完了した当日分の指示書** を回収する。
- 常に現場には **最新の計画が記載された指示書** だけが置かれる状態を保つ。

### 3.2 手順

1. 集計担当者が **新規発行した指示書**（第 2 章）を各成型ラインへ配布する。
2. 現場担当者は、**前日までに使用し記入済みの指示書** を集計担当者へ返却する。
3. 返却時に確認する項目（例）：
   - ・生産数の手書き記入が読み取れるか
4. 回収した指示書は、PLC データ・HT 実績と突合するための **現場記録** として保管する。

![現場での指示書交換](./images/FormingInstructionManual/08_site_exchange.png)

### 3.3 注意事項

- 指示書の交換は **シフト開始前** または **日替わり時** に実施するのが望ましいです。
- 新しい指示書を渡す前に、旧指示書から **未反映の実績が無いか** 現場リーダーに確認してください。
- 計画変更が当日中に入った場合は、再発行した指示書で **差し替え** ます。

## 4. PLC 収集データ印刷

### 4.1 概要

成型ラインの PLC システムが収集した生産データ（計画数・実績数等）を **帳票として印刷** し、HT 登録や ERP 照合の元資料とします。

### 4.2 参照資料

詳細なボタン操作・画面遷移は、社内資料 **「PLCシステム_計画数アップロード_操作手順.pdf」** に従ってください。

社内資料の保存場所：**社内共有フォルダ** ⇒ **生産管理部** ⇒ **マニュアル**

### 4.3 実施タイミング

- 通常は **生産終了後**、または **日次締め前** に実施します。
- 第 3 章で回収した現場記入指示書と **同一日・同一設備・同一製品** で突合できるよう、日付とライン名を必ず確認してください。

### 4.4 確認ポイント

| 確認項目 | 説明 |
|----------|------|
| 対象日 | 照合対象の生産日と一致しているか |
| 設備／ライン | 成型機ごとにデータが分かれているか |
| 計画数・実績数 | 0 や異常値がないか |
| 品種切替 | 段替直後の品種が指示書と一致しているか |

![PLC 収集データ印刷](./images/FormingInstructionManual/09_plc_print.png)

## 6. 実績照合（生産実績管理）

PLC 印刷データ・HT 登録データ・現場指示書の記入内容を、ERP 上の在庫取引実績と **照合** します。

### 6.1 画面へのアクセス

1. **ERP管理メニュー** → **生産管理** → **生産実績** → **生産実績管理**
2. 画面タイトル **「生産実績管理」**

![メニューから生産実績管理](./images/FormingInstructionManual/11_menu_actual_management.png)

### 6.2 照合の手順

1. **検索条件を設定する**
   - **製品名**：照合対象製品を選択（任意）
   - **設備**：成型機を選択（任意）
   - **生産日**：対象日を範囲指定。クイックボタン **前日／今日／翌日** も利用可

2. **工程別フィルタで「成型」を選択する**
   - 「工程別フィルタ」タブから **成型** 工程を選びます。
   - サマリカード（件数・数量等）と **取引タイプ内訳** を確認します。

   ![生産実績管理 成型タブ](./images/FormingInstructionManual/12_actual_management_filter.png)

3. **取引ログ一覧で照合する**
   - 一覧列：取引日時、工程、タイプ、製品 CD／名、数量、保管場所、設備名 等
   - **PLC 印刷値・HT 登録値・指示書記入値** と ERP の **数量** を突合します。
   - 差異がある行は原因を確認（二重登録、未登録、品種違い 等）。

4. **差異の修正**（必要な場合）
   - 行右端 **「編集」**：数量等を修正（取引ログ編集ダイアログ）
   - **「削除」**：誤登録の削除
   - 修正後 **「再取得」** で最新状態を確認

5. **帳票出力**（任意）
   - **取引ログ印刷**：一覧を印刷

![実績照合の例](./images/FormingInstructionManual/13_actual_reconcile.png)

### 6.3 照合チェックリスト

| # | 確認内容 | OK |
|---|----------|-----|
| 1 | 対象日の成型実績件数が PLC／HT と同程度か | ☐ |
| 2 | 設備名・製品 CD が指示書と一致しているか | ☐ |
| 3 | 数量の合計が指示書・PLC の合計と一致するか | ☐ |

## 7. 収集実績のデータベース更新（全部一括更新）

照合が完了したら、収集済み実績を **データベース管理集計テーブル** へ反映します。

### 7.1 画面へのアクセス

1. **ERP管理メニュー** → **生産管理** → **生産計画** → **生産データ管理**
2. 画面タイトル **「生産データ管理」**（副題：受注・実績・在庫を一元管理）


> メニュー上は「生産計画」の下に「生産データ管理」があります。画面上部の **「各種更新機能」** ドロップダウンから一括更新を実行します。

![メニューと生産データ管理](./images/FormingInstructionManual/14_menu_data_management.png)

### 7.2 「全部一括更新」の操作手順

1. 画面右上 **「各種更新機能」** ボタン（▼付き）をクリックします。
2. ドロップダウンから **「全部一括更新」** を選択します。
3. 確認ダイアログ **「全部一括更新確認」** が表示されます。以下の順序で更新されます：

   | 順序 | 処理名 |
   |------|--------|
   | 1 | 受注データ更新 |
   | 2 | 実績データ更新 |
   | 3 | 不良データ更新 |
   | 4 | 廃棄データ更新 |
   | 5 | 保留データ更新 |
   | 6 | 計画データ更新 |
   | 7 | 在庫・推移・安全在庫更新 |

4. **「一括更新開始」** をクリックします。
5. 進捗ダイアログで処理完了を待ちます（時間がかかる場合があります）。
6. 他端末で同時実行中の場合は **「他の端末で一括更新が実行中…」** と表示され、完了後に再実行してください。

![全部一括更新](./images/FormingInstructionManual/15_all_batch_update.png)

### 7.3 実行タイミングの目安

- 第 6 章の照合・修正が **すべて完了した後** に 1 回実行する。

## 8. データベース調整（在庫 TAB）

一括更新後も、在庫数に **微差** がある場合は、生産データ管理画面の **在庫 TAB** から直接修正します。

### 8.1 在庫 TAB を開く

 **生産データ管理** 画面の下部タブ列から **「在庫」**（📦 アイコン）をクリックします。

![生産データ管理：下部タブ列の「在庫」位置](./images/FormingInstructionManual/16_inventory_tab.png)

### 8.2 セルのダブルクリック修正

1. 修正したい **日付行** と **工程列**（例：成型実績、成型不良、成型在庫 等）の交差セルを **ダブルクリック** します。
2. **「在庫取引ログ入力」** ダイアログが開きます。
   - 上部に日付・製品 CD・製品名・工程が表示されます。
3. 工程に応じた入力欄に数量を入力します。

   | 工程種別 | 入力項目 |
   |----------|----------|
   | 一般工程（成型 等） | 実績、不良、廃棄、保留 |
   | 倉庫（KT13） | 入庫、出庫、廃棄、保留 |
   | 外注（KT15） | 入庫、廃棄 |

4. **登録** を実行すると `データベース管理表` に取引が追加され、後続の在庫再計算で反映されます。
5. 大幅な修正を多数行う場合は、再度 **全部一括更新** または **在庫・推移更新** の実行を検討してください。

![在庫 TAB ダブルクリック](./images/FormingInstructionManual/16_inventory_tab_dblclick.png)

### 8.3 注意事項

- 製品が当該工程ルートに含まれない場合、「製品は工程に属していません」と表示されます。
- 基本情報列（日付・製品 CD 等）のダブルクリックではダイアログは開きません。

## 9. APS 再計算と生産残の修正

実績反映後、APS 側の成型計画を **ライン順に再計算** し、計画と実績の **「残」** を確認・修正します。

### 9.1 成型計画一覧で再計算

1. **APS管理メニュー** → **生産計画一覧** → **成型計画一覧**
2. 画面タイトル **「成型計画一覧」**
3. **期間** と **工程** を指定し、一覧／ガントを表示します。
4. 右上 **「成型ライン順で再計算」** ボタン（オレンジ色）をクリックします。
5. 確認メッセージを読み、実行を承認します。
   - 過去日の日別計画は固定されます。
   - 本日分は実績がある場合のみ固定、ない場合は当日以降を再計算します。
6. **再計算進捗** バーで設備ごとの進行を確認します。

![成型ライン順で再計算ボタン](./images/FormingInstructionManual/18_replan_button.png)

### 9.2 生産残の確認（要修正判定）

再計算完了後、日別ガントおよび一覧表の **「翌日の計画残数」** を確認します。

**修正が必要な典型パターン：**

| パターン | 判定基準 | リスク |
|----------|----------|--------|
| A. 少量残 | **残 &lt; 50 本** 程度でロットが実質生産完了した | 次計画に残数が持ち越されず|
| B. 次品種着手済み | **残 &gt; 0** なのに **次の品種の計画／実績が既に開始** している | 同一設備で 2 品種が重なった計画になる |

![残数確認（ガント／一覧）](./images/FormingInstructionManual/19_remain_check_gantt.png)

### 9.3 修正方法（成型計画作成 — 合計(本)）

残数を解消するには **APS管理メニュー** → **生産計画作成** → **成型計画作成** を開き、対象ラインの **計画一覧** テーブルで **「合計(本)」** を修正します。

| 項目 | 値 |
|------|-----|
| URL パス | `/aps/planning` |

#### 操作手順

1. 対象 **設備（ライン）** を選択し、**計画一覧** を表示します。
2. 修正対象行の **「合計(本)」** セルを **ダブルクリック** します（ツールチップ：「ダブルクリックで数量を変更」）。
3. 数値を入力し **Enter** またはフォーカスアウトで保存します。
   - **1 以上の整数** が必須です。
4. 増加時は末尾追記、減少時はライン再計算が内部で実行されます。

![合計(本)の編集](./images/FormingInstructionManual/20_forming_planning_total_qty.png)

#### 修正例（ケーススタディ）

**状況（パターン A：少量残）**

| 項目 | 値 |
|------|-----|
| 設備 | 成型 03 号機 |
| 製品 | GC7 CTR |
| 合計(本)（修正前） | 1,200 本 |
| 実績累計 | 1,165 本 |
| 残 | **35 本**（50 未満） |

現場ではロット完了扱いだが、システム上 35 本残っている。

**対応**

1. 成型計画作成で GC7 CTR の行を開く。
2. **合計(本)** を `1200` → **`1165`** に変更（実績に合わせて残を 0 にする）。
3. 保存後、成型計画一覧で **再度「成型ライン順で再計算」** を実行。
4. ガント上で **残 = 0** または残表示が消えることを確認。

---

**状況（パターン B：次品種着手済み）**

| 項目 | 値 |
|------|-----|
| 設備 | 成型 05 号機 |
| 前品種 DEF-5678 | 合計 800 本、実績 780 本、**残 20 本** |
| 次品種 GHI-9012 | 同日午後から計画／実績開始済み |

**対応**

1. DEF-5678 の **合計(本)** を `800` → **`780`** に修正（残 20 を計画上解消）。
2. または現場が 20 本追加生産する場合は **合計(本)** を維持し、次品種 GHI-9012 の **開始日指定** を見直す（別途計画担当が判断）。
3. 再計算後、**同一設備で品種が重ならない** ことをガントで確認。

## 10. 段取表印刷と現場掲示

残数修正と **2 回目の APS 再計算**（第 9.3 節末尾）が完了したら、翌日および **翌稼働日（翌日+1 稼働日）** の段替予定を現場へ掲示します。

### 10.1 再度 APS 再計算

1. **成型計画一覧** で **「成型ライン順で再計算」** を **もう一度** 実行します（第 9.1 節と同じ）。
2. 計画データが最新になったことを確認してから、次の段取表印刷に進みます。

### 10.2 成型指示画面で段取表を印刷

1. **ERP管理メニュー** → **生産管理** → **生産指示** → **成型指示** を開きます。
2. **生産日** を **翌日** に設定します（第 2 章と同様）。
   - システムは智能日付範囲（前日・当日・翌稼働日 等）を自動算出し、**翌日および翌稼働日** の計画を段取表に含めます。
   - 金曜日基準の場合は翌週月曜・火曜、土曜日基準の場合は翌週月〜水 等も考慮されます。
3. 次のいずれかで印刷します。

   | ボタン | 説明 |
   |--------|------|
   | **段取表プレビュー** | プレビュー画面で内容を確認・編集してから **印刷** |
   | **段取表印刷** | 編集せずそのまま印刷（A4 横向き） |

4. 印刷物タイトル：**「成型生産計画段替予定表」**
   - 設備ごとの段替順、品種、計画数、予定稼働時間、操業度 等が記載されます。

![段取表印刷ボタン](./images/FormingInstructionManual/21_setup_schedule_print.png)  
![段替予定表（印刷サンプル）](./images/FormingInstructionManual/22_setup_schedule_sample.png)

### 10.3 段取表プレビューで編集する場合

1. **段取表プレビュー** をクリックします。
2. ダイアログ内の表で、必要に応じて **操業度(進捗)** 等を修正できます。
3. ダイアログ下部 **「印刷」** で確定印刷します。

### 10.4 現場掲示

1. 印刷した **成型生産計画段替予定表** を、成型工程工程の掲示板に掲出します。
2. 掲示対象：**翌日分** および **翌稼働日（翌日+1 稼働日）** が一覧で分かる部分。
3. 古い段替表は必ず **剥がして差し替え**、複数版の混在を防ぎます。
4. 計画変更が入った場合は、再計算 → 再印刷 → **再掲示** をセットで実施します。

## 11. よくあるメッセージと対処

| メッセージ | 画面 | 対処 |
|------------|------|------|
| 印刷対象の設備がありません | 成型指示 | 成型設備マスタ・計画データを確認。データ更新を実行 |
| QRコード読み込み中… | 成型指示 | ネットワーク確認。完了まで待つか × で中止 |
| 印刷する計画データがありません | 成型指示（段取表） | 生産日・APS 計画を確認。再計算後に再試行 |
| 該当データがありません | 生産実績管理 | フィルタ条件（日付・工程・製品）を見直し |
| 他の端末で一括更新が実行中… | 生産データ管理 | 数分待ってから再実行 |
| 合計(本)は 1 以上の整数を入力してください | 成型計画作成 | 正の整数を入力 |
| 製品は工程に属していません | 生産データ管理 | 製品ルート・工程 CD を確認 |
| 再計算に失敗しました | 成型計画一覧 | 期間・工程選択を確認。失敗設備名を確認して再実行 |
